# Level 1: 트리를 직접 만들어본다

## 의사결정 트리란

사람이 판단하는 과정을 그대로 모방한 것이다.

```
"이 이메일이 스팸인가?"

질문 1: "무료"라는 단어가 있는가?
  ├─ Yes → 질문 2: 발신자가 연락처에 있는가?
  │         ├─ Yes → 정상
  │         └─ No  → 스팸
  └─ No  → 질문 3: 첨부파일이 있는가?
            ├─ Yes → 스팸
            └─ No  → 정상
```

핵심 질문: **어떤 순서로 질문해야 가장 빠르게 답에 도달하는가?**

"무료"를 먼저 물어보는 게 나은가, "첨부파일"을 먼저 물어보는 게 나은가? 이 선택을 수학적으로 하는 것이 의사결정 트리 알고리즘이다.

## 예제 데이터: 테니스를 칠 것인가?

```
날씨     | 온도  | 습도  | 바람  | 테니스?
─────────|───────|───────|───────|────────
맑음     | 더움  | 높음  | 약함  | No
맑음     | 더움  | 높음  | 강함  | No
흐림     | 더움  | 높음  | 약함  | Yes
비       | 보통  | 높음  | 약함  | Yes
비       | 추움  | 보통  | 약함  | Yes
비       | 추움  | 보통  | 강함  | No
흐림     | 추움  | 보통  | 강함  | Yes
맑음     | 보통  | 높음  | 약함  | No
맑음     | 추움  | 보통  | 약함  | Yes
비       | 보통  | 보통  | 약함  | Yes
맑음     | 보통  | 보통  | 강함  | Yes
흐림     | 보통  | 높음  | 강함  | Yes
흐림     | 더움  | 보통  | 약함  | Yes
비       | 보통  | 높음  | 강함  | No

전체 14건: Yes 9건, No 5건
```

4개의 특성(날씨, 온도, 습도, 바람) 중 **어떤 것을 첫 번째 분할 기준으로 쓸 것인가?**

## 엔트로피 — 불확실성을 숫자로 재기

엔트로피(Entropy)는 "얼마나 뒤섞여 있는가"를 측정한다.

$$H(S) = -\sum_{i=1}^{c} p_i \log_2 p_i$$

직관:

```
상자 안에 공이 들어있다.

상자 A: 빨강 10개, 파랑 0개
  → 꺼내면 100% 빨강. 불확실성 없음.
  → H = -1·log₂(1) = 0

상자 B: 빨강 5개, 파랑 5개
  → 꺼내면 뭐가 나올지 모름. 최대 불확실성.
  → H = -0.5·log₂(0.5) - 0.5·log₂(0.5) = 0.5 + 0.5 = 1.0

상자 C: 빨강 9개, 파랑 1개
  → 거의 빨강이지만 약간 불확실.
  → H = -0.9·log₂(0.9) - 0.1·log₂(0.1) = 0.137 + 0.332 = 0.469
```

```
H
1.0 ─           ●
    |          ╱ ╲
    |         ╱   ╲
0.5 ─       ╱     ╲
    |      ╱       ╲
    |     ╱         ╲
  0 ●───────────────────●
    0        0.5        1.0    ← p (양성 비율)

p=0이나 p=1이면 H=0 (확실)
p=0.5이면 H=1 (최대 불확실)
```

## 테니스 데이터의 엔트로피 직접 계산

전체 14건: Yes 9건, No 5건

$$H(\text{전체}) = -\frac{9}{14}\log_2\frac{9}{14} - \frac{5}{14}\log_2\frac{5}{14}$$

```
계산:
  9/14 = 0.643
  5/14 = 0.357

  -0.643 × log₂(0.643) = -0.643 × (-0.638) = 0.410
  -0.357 × log₂(0.357) = -0.357 × (-1.486) = 0.531

  H = 0.410 + 0.531 = 0.940
```

전체 엔트로피는 **0.940**이다. 1에 가까우므로 꽤 뒤섞여 있다.

## 정보 이득 — 어떤 특성이 가장 잘 나누는가

정보 이득(Information Gain) = **분할 전 엔트로피 - 분할 후 엔트로피**

분할 후 엔트로피가 많이 떨어질수록 그 특성이 좋다.

**"날씨"로 분할하면:**

```
맑음(5건): Yes 2, No 3  → H = -2/5·log₂(2/5) - 3/5·log₂(3/5) = 0.971
흐림(4건): Yes 4, No 0  → H = 0  (모두 Yes → 완벽히 정리!)
비(5건):   Yes 3, No 2  → H = -3/5·log₂(3/5) - 2/5·log₂(2/5) = 0.971

분할 후 가중 평균 엔트로피:
  = (5/14) × 0.971 + (4/14) × 0 + (5/14) × 0.971
  = 0.347 + 0 + 0.347
  = 0.694

IG(날씨) = H(전체) - H(분할 후) = 0.940 - 0.694 = 0.246
```

**"바람"으로 분할하면:**

```
약함(8건): Yes 6, No 2  → H = -6/8·log₂(6/8) - 2/8·log₂(2/8) = 0.811
강함(6건): Yes 3, No 3  → H = -3/6·log₂(3/6) - 3/6·log₂(3/6) = 1.000

분할 후 가중 평균:
  = (8/14) × 0.811 + (6/14) × 1.000
  = 0.463 + 0.429
  = 0.892

IG(바람) = 0.940 - 0.892 = 0.048
```

**4개 특성의 정보 이득 비교:**

```
특성      IG       해석
──────────────────────────────
날씨     0.246    ← 가장 높음! 첫 분할 기준
습도     0.151
바람     0.048
온도     0.029    ← 가장 낮음. 별로 도움 안 됨
```

**날씨의 정보 이득이 가장 크므로 첫 번째 분할 기준으로 선택한다.**

특히 "흐림"은 엔트로피가 0이다 — 흐린 날은 100% 테니스를 친다. 더 이상 분할할 필요가 없다(리프 노드).

## 트리 완성

```
                    [날씨?]
                   /   |   \
                맑음  흐림   비
                 /     |     \
          [습도?]    Yes    [바람?]
          /    \            /    \
       높음   보통        약함   강함
        |      |          |      |
       No    Yes        Yes     No
```

이것이 **ID3 알고리즘**이 만든 트리다. 각 노드에서 정보 이득이 가장 큰 특성을 선택하여 재귀적으로 분할한다.

## 지니 불순도 — 엔트로피의 대안

scikit-learn의 기본 설정은 엔트로피 대신 **지니 불순도(Gini Impurity)**를 쓴다.

$$Gini(S) = 1 - \sum_{i=1}^{c} p_i^2$$

```
같은 상자 예제로:

상자 A: 빨강 10개, 파랑 0개
  Gini = 1 - 1² = 0

상자 B: 빨강 5개, 파랑 5개
  Gini = 1 - 0.5² - 0.5² = 1 - 0.25 - 0.25 = 0.5

상자 C: 빨강 9개, 파랑 1개
  Gini = 1 - 0.9² - 0.1² = 1 - 0.81 - 0.01 = 0.18
```

```
엔트로피 vs 지니:

     엔트로피              지니
     최대 1.0              최대 0.5 (이진)
     로그 연산 필요         곱셈만 (계산 빠름)
     정보이론 기반           확률적 해석
     결과는 대부분 비슷      결과는 대부분 비슷
```

실전에서 둘의 차이는 거의 없다. 계산이 빠른 지니가 기본값인 경우가 많다.

## 연속형 특성은 어떻게?

ID3는 범주형만 다룬다. 연속형(나이, 소득 등)은 **CART 알고리즘**이 처리한다.

```
데이터: 나이가 [25, 30, 35, 40, 45]이고
       레이블이 [No, No, Yes, Yes, Yes]

모든 분할점 후보를 시도:
  나이 ≤ 27.5?  → {25} vs {30,35,40,45}  → 지니 계산
  나이 ≤ 32.5?  → {25,30} vs {35,40,45}  → 지니 계산  ← 최적!
  나이 ≤ 37.5?  → {25,30,35} vs {40,45}  → 지니 계산

"나이 ≤ 32.5"가 지니 감소가 가장 크면 이 기준으로 분할.
→ 결정 경계가 축에 평행한 직선이 된다.
```

# Level 2: 과적합과 그 해결

## 트리의 치명적 약점: 과적합

트리를 제한 없이 키우면 **학습 데이터를 완벽하게 외운다.**

```
극단적인 경우: 모든 샘플에 대해 리프 노드 하나씩
  → 학습 정확도: 100%
  → 테스트 정확도: 처참

학습 데이터에 노이즈(측정 오류, 이상치)가 있다면:
  트리가 그 노이즈까지 학습해버린다.

  예: "35세, 소득 5000만원, 부양가족 2명 → 대출 승인"이
      실은 데이터 입력 오류였는데, 트리가 이 패턴을 규칙으로 학습
```

```
                학습 정확도          테스트 정확도
트리 깊이 3:      85%                   82%     ← 적절
트리 깊이 5:      92%                   84%     ← 괜찮음
트리 깊이 10:     98%                   78%     ← 과적합 시작!
트리 깊이 20:    100%                   65%     ← 심각한 과적합
                  ↑                      ↑
           외우기는 잘함            새 데이터에 무용지물
```

## 가지치기 (Pruning)

**사전 가지치기 (Pre-pruning):** 트리가 자라는 도중에 멈춘다.

```python
# scikit-learn
from sklearn.tree import DecisionTreeClassifier

tree = DecisionTreeClassifier(
    max_depth=5,           # 최대 깊이 제한
    min_samples_split=10,  # 분할하려면 최소 10개 샘플
    min_samples_leaf=5,    # 리프 노드에 최소 5개 샘플
    max_features='sqrt'    # 분할 시 고려할 특성 수 제한
)
```

**사후 가지치기 (Post-pruning):** 트리를 끝까지 키운 뒤, 아래에서부터 가지를 잘라낸다.

```
가지치기 전:
        [A?]
       /    \
    [B?]    [C?]
   / \     / \
  Y   N  [D?]  Y
         / \
        Y   N

가지치기: [D?] 노드를 잘라낸다.
  → [D?] 아래 Y, N 중 다수결로 대체

가지치기 후:
        [A?]
       /    \
    [B?]    [C?]
   / \     / \
  Y   N   Y   Y    ← [D?] 노드가 Y로 대체됨

검증 데이터에서 정확도가 같거나 높아지면 가지치기 유지.
```

**하지만 가지치기만으로는 부족하다.** 트리 하나는 근본적으로 **분산(variance)이 높다** — 데이터가 조금만 바뀌어도 완전히 다른 트리가 만들어진다.

```
같은 데이터에서 1건만 빼도:

원래:          [날씨?]              1건 제거 후:    [습도?]
              /  |  \                              /     \
           맑음 흐림  비                         높음    보통
            ...                                  ...
→ 첫 분할 기준이 아예 바뀐다!
```

이 문제를 해결하는 것이 **앙상블**이다.

# Level 3: 앙상블 — 약한 트리를 모아 강한 모델로

트리 하나는 과적합에 취약하고 불안정하다. 이를 해결하는 것이 앙상블이다. 각 방법은 별도 포스트에서 깊이 다룬다.

## Bias-Variance 분해 — 왜 앙상블이 필요한가

모델의 오차는 세 가지로 분해된다:

$$\text{총 오차} = \text{Bias}^2 + \text{Variance} + \text{Noise}$$

```
Bias (편향):  모델이 너무 단순해서 패턴을 못 잡는 오차
Variance (분산): 데이터가 바뀌면 모델이 크게 흔들리는 오차
Noise: 데이터 자체의 노이즈. 어떤 모델로도 줄일 수 없다.

깊은 트리 1개:  낮은 Bias, 높은 Variance → 과적합
얕은 트리 1개:  높은 Bias, 낮은 Variance → 과소적합
```

앙상블은 이 딜레마를 해결한다:

```
배깅 (랜덤 포레스트):  깊은 트리 100개의 평균 → Variance를 줄인다
부스팅 (XGBoost):     얕은 트리를 순차적으로 → Bias를 줄인다
스태킹:               서로 다른 모델을 조합 → 둘 다 줄인다
```

## 앙상블 방법 시리즈

각 방법의 상세한 원리, 수학, 코드는 별도 포스트에서 다룬다:

### 1. [랜덤 포레스트](/random-forest/) — 배깅

```
핵심: 다양한 트리를 만들어 다수결로 예측

부트스트랩(복원 추출) + 특성 랜덤 선택
→ 트리들이 "서로 다른 실수"를 하게 만듦
→ 평균 내면 개별 오차가 상쇄

트리 1개 오류율 35% → 100개 다수결 오류율 0.07%

> 이 계산은 100개 트리의 오류가 **독립**이라고 가정한다. 실제 랜덤 포레스트에서는 bootstrap 샘플이 겹치므로 트리 간 상관관계(ρ)가 존재하고, 실제 오류율은 0.07%보다 높다. 독립 가정은 다수결의 이론적 이점을 설명하기 위한 것이며, 실전 성능과는 다르다.

상세: Bias-Variance 분산 공식(상관계수 ρ 포함), OOB 검증,
     특성 중요도(MDI vs Permutation), 하이퍼파라미터 튜닝
```

### 2. [그래디언트 부스팅과 XGBoost](/gradient-boosting-xgboost/) — 부스팅

```
핵심: 이전 모델이 틀린 것을 다음 모델이 보완

AdaBoost: 틀린 샘플의 가중치를 올려서 집중
그래디언트 부스팅: 잔차(오차)를 다음 트리가 학습
XGBoost: 2차 근사 + 정규화로 최적화

실제 값 [10,20,30,40] → 트리 1 → 잔차 → 트리 2 → 잔차 → ...

상세: AdaBoost 수치 예제, 잔차 학습 전 과정,
     XGBoost 2차 테일러 전개, Gain 공식, 히스토그램 분할
```

### 3. [LightGBM과 CatBoost](/lightgbm-catboost/) — 최신 부스팅

```
LightGBM: Leaf-wise 성장 + GOSS(샘플링) → XGBoost보다 3배 빠름
CatBoost: 범주형 특성을 원-핫 없이 직접 처리 → 범주형 많을 때 최적

              XGBoost      LightGBM      CatBoost
속도           빠름        매우 빠름       보통
범주형 처리    원-핫 필요   원-핫 필요     직접 처리
적합한 상황    범용         대용량         범주형 많을 때

상세: Leaf-wise vs Level-wise, GOSS 알고리즘,
     EFB, Ordered Target Statistics, Oblivious Trees
```

### 4. [스태킹과 앙상블 전략](/stacking-ensemble/) — 모델 조합

```
핵심: 서로 다른 유형의 모델을 메타 모델이 조합

Level 0: RF, XGBoost, LR, KNN → 각각 예측
Level 1: 메타 모델이 예측들을 입력받아 최종 결정

K-fold로 과적합 방지, 블렌딩 vs 스태킹 비교
```

## 실전 요약

```
1단계: 랜덤 포레스트로 베이스라인 → 튜닝 없이도 꽤 좋은 성능
2단계: LightGBM으로 성능 끌어올리기 → 하이퍼파라미터 튜닝
3단계: (대회) 스태킹 → 실무에서는 복잡도 대비 이득이 작음
4단계: 해석 필요하면 SHAP → 어떤 특성이 얼마나 기여했는지
```

## 이 글에서 가져가야 할 것

```
Level 1:
  엔트로피 = 뒤섞임의 정도 (0이면 깨끗, 1이면 최대 혼란)
  정보 이득 = 분할 전 엔트로피 - 분할 후 엔트로피
  가장 큰 정보 이득을 주는 특성을 먼저 쓴다

Level 2:
  트리 하나는 과적합에 취약하고 불안정하다
  가지치기로 완화하지만 근본적 해결은 아니다

Level 3:
  배깅(RF) = Variance를 줄인다. 부스팅(XGBoost) = Bias를 줄인다.
  RF로 시작 → LightGBM으로 최적화 → 필요시 스태킹
```

앙상블 시리즈: [랜덤 포레스트](/random-forest/) → [그래디언트 부스팅/XGBoost](/gradient-boosting-xgboost/) → [LightGBM/CatBoost](/lightgbm-catboost/) → [스태킹](/stacking-ensemble/)

다음 글: [정규화](/regularization/)에서 과적합 방지를 베이지안 관점([베이즈 정리](/bayes-theorem/)의 MAP)으로 다시 만난다.
