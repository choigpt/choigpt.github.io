# Level 1: LightGBM -- 빠른 그래디언트 부스팅

[이전 글](/gradient-boosting-xgboost/)에서 XGBoost가 그래디언트 부스팅을 2차 근사와 정규화로 최적화했음을 봤다. LightGBM(Light Gradient Boosting Machine)은 Microsoft Research에서 2017년에 발표한 프레임워크로, XGBoost와 같은 원리를 쓰되 **학습 속도를 극적으로 끌어올린다.** 핵심 아이디어 세 가지를 하나씩 본다.

## Leaf-wise vs Level-wise 트리 성장

XGBoost를 포함한 대부분의 트리 알고리즘은 **Level-wise**로 트리를 키운다. 같은 깊이(레벨)에 있는 모든 노드를 한 번에 분할한다.

```
Level-wise 성장 (XGBoost 기본):

깊이 0:           [root]
                  /      \
깊이 1:        [A]        [B]         <- 이 레벨 전부 분할
                / \        / \
깊이 2:      [C]  [D]   [E]  [F]     <- 이 레벨도 전부 분할
              / \  / \   / \  / \
깊이 3:     ... ... ... ... ... ...   <- 또 전부 분할

문제: E와 F의 정보 이득이 거의 0이어도 분할한다.
     계산 낭비 + 과적합 위험.
```

LightGBM은 **Leaf-wise**(Best-first)로 트리를 키운다. 현재 존재하는 모든 리프 중에서 **정보 이득이 가장 큰 리프 하나만** 분할한다.

```
Leaf-wise 성장 (LightGBM 기본):

Step 1:           [root]
                  /      \
                [A]      [B]

  A의 이득: 0.85,  B의 이득: 0.32
  -> A를 분할 (이득이 더 크니까)

Step 2:           [root]
                  /      \
               [A]       [B]
              /    \       |
            [C]    [D]    (대기)

  C의 이득: 0.12,  D의 이득: 0.91,  B의 이득: 0.32
  -> D를 분할

Step 3:           [root]
                  /      \
               [A]       [B]
              /    \       |
            [C]    [D]    (대기)
                  /    \
                [E]    [F]

  C의 이득: 0.12,  E의 이득: 0.45,  F의 이득: 0.28,  B의 이득: 0.32
  -> E를 분할

Step 4:           [root]
                  /      \
               [A]       [B]
              /    \       |
            [C]    [D]    (대기)
                  /    \
                [E]    [F]
               /   \
             [G]   [H]
```

## 왜 Leaf-wise가 더 빠른가

같은 수의 리프 노드를 만든다고 하자. 리프 8개 기준:

```
Level-wise (리프 8개):
  깊이 3의 완전 이진 트리 = 7번 분할, 깊이 3
  내부 노드 7개 전부에 대해 최적 분할점을 탐색해야 한다.
  이 중 일부는 이득이 거의 없는 쓸모없는 분할이다.

        [root]                    <- 분할 1회
       /      \
     [A]      [B]                 <- 분할 2회
    / \      / \
  [C] [D]  [E] [F]               <- 분할 4회 (이득 작아도 전부 분할)
  / \ / \  / \ / \
 리프 8개                          총 7회 분할

Leaf-wise (리프 8개):
  7번 분할하지만 매번 이득이 가장 큰 리프만 분할.
  같은 리프 수에서 총 손실 감소량이 더 크다.

        [root]
       /      \
     [A]      [B]
    / \        |
  [C] [D]     [E]
      / \    /   \
    [F] [G] [H]  [I]

  비대칭이지만, 7번의 분할 각각이 최대 이득을 준다.
  -> 같은 리프 수로 더 정확한 모델.
  -> 이득 없는 분할을 건너뛰므로 학습도 빠르다.
```

핵심: Leaf-wise는 **같은 리프 수에서 더 낮은 손실**을 달성한다. 같은 정확도를 원한다면 **더 적은 리프로** 충분하므로 학습이 빨라진다.

## Leaf-wise의 과적합 위험

Leaf-wise의 약점은 **비대칭 트리가 매우 깊어질 수 있다**는 것이다.

```
극단적인 Leaf-wise 트리:

          [root]
         /      \
       [A]      리프
      /    \
    [B]    리프
   /    \
 [C]    리프
 /  \
...

-> 한쪽으로 치우쳐서 깊이가 매우 깊어진다.
-> 깊은 경로 = 매우 구체적인 규칙 = 과적합
```

이를 방지하려면 **num_leaves**를 직접 제한해야 한다.

```
num_leaves = 31 (기본값):
  트리가 아무리 비대칭이어도 리프가 31개를 넘지 못한다.
  깊이 제한(max_depth)보다 더 직접적인 복잡도 제어.

num_leaves vs max_depth 관계:
  Level-wise에서 max_depth=5 -> 리프 최대 2^5 = 32개
  Leaf-wise에서 num_leaves=31 -> 비슷한 복잡도

  실전 규칙: num_leaves < 2^(max_depth)
  예: max_depth=7이면 num_leaves는 100 정도가 적절 (2^7=128보다 작게)
```

## 히스토그램 기반 분할

XGBoost도 히스토그램을 쓰지만, LightGBM은 처음부터 히스토그램 중심으로 설계되었다.

연속형 특성의 모든 가능한 분할점을 시도하면 계산량이 데이터 크기에 비례한다. 히스토그램은 이를 상수 시간으로 바꾼다.

```
원래 방식 (exact greedy):
  "나이" 특성에 100만 개의 서로 다른 값이 있다면
  -> 100만 개의 분할점 후보를 전부 시도
  -> O(n) 시간

히스토그램 방식:
  Step 1: 100만 개의 값을 256개 구간(bin)으로 미리 묶는다.

  연속 값:  18.2, 18.5, 19.1, 19.3, ..., 79.8, 80.0
                       |
                       v
  구간 할당: bin 0: [18.0-18.24)  -> 1523건
            bin 1: [18.24-18.48) -> 1487건
            bin 2: [18.48-18.72) -> 1512건
            ...
            bin 255: [79.76-80.0] -> 892건

  Step 2: 분할점 후보 = 256개 구간 경계
  -> O(256) = O(1) 시간 (데이터 크기에 무관!)
```

구간 경계를 정하는 방법:

```
1. 등폭 분할 (Equal-width):
   (최대값 - 최소값) / 256으로 균등하게 나눈다.
   단점: 데이터가 편향되면 빈 구간이 많아진다.
   예: 소득이 [0, 10억]인데 99%가 [0, 1억] 사이
       -> 대부분의 구간이 비어있음

2. 등빈도 분할 (Equal-frequency, LightGBM 기본):
   각 구간에 데이터가 비슷한 수만큼 들어가도록 나눈다.
   예: 100만 건 / 256 = 약 3,906건씩
   -> 밀집 구간은 구간을 좁게, 희소 구간은 구간을 넓게
   -> 정보 손실 최소화

   실제 데이터:
   소득 [0, 10억], 99%가 [0, 1억]
   -> [0, 1억] 구간에 약 253개 구간 배정 (촘촘하게)
   -> [1억, 10억] 구간에 약 3개 구간 배정 (넓게)
```

히스토그램의 추가 이점 -- **히스토그램 차분(histogram subtraction)**:

```
부모 노드의 히스토그램은 이미 계산되어 있다.
왼쪽 자식의 히스토그램을 계산하면,
오른쪽 자식 = 부모 - 왼쪽  (단순 빼기)

부모: [100, 80, 120, 90, ...]  (각 구간의 그래디언트 합)
왼쪽: [60, 30, 70, 50, ...]
오른쪽: [40, 50, 50, 40, ...]  <- 뺄셈으로 즉시 구함!

-> 자식 노드 하나만 계산하면 나머지는 공짜.
-> 작은 쪽 자식만 계산하면 더 빠르다 (데이터가 적으니까).
```

## Exclusive Feature Bundling (EFB)

고차원 희소(sparse) 데이터에서 LightGBM이 빠른 또 다른 이유다.

```
상황: 원-핫 인코딩된 특성이 많다.

특성 A: [1, 0, 0, 0, 0, 0, ...]  <- "서울"
특성 B: [0, 1, 0, 0, 0, 0, ...]  <- "부산"
특성 C: [0, 0, 1, 0, 0, 0, ...]  <- "대구"

관찰: A=1이면 반드시 B=0, C=0 (상호 배타적!)
     -> 하나의 특성으로 묶을 수 있다.

EFB 결과:
특성 AB_C: [1, 2, 3, 0, 0, 0, ...]
  값 1 = 원래 A가 1이었음
  값 2 = 원래 B가 1이었음
  값 3 = 원래 C가 1이었음

3개 특성이 1개로 줄었다.
-> 히스토그램 계산 횟수도 1/3로 감소.
```

EFB 알고리즘:

```
1. 특성 간 "충돌"을 계산한다.
   충돌 = 두 특성이 동시에 0이 아닌 샘플의 수
   A와 B의 충돌: 0건 (완전 상호 배타)
   A와 D의 충돌: 50건 (약간 겹침)

2. 충돌이 적은 특성들을 하나의 번들로 묶는다.
   허용 충돌 비율 = max_conflict_rate (기본 0)
   0이면 완전 상호 배타만 묶음, 약간 올리면 더 공격적으로 묶음.

3. 번들 내 특성들의 값을 오프셋으로 구분한다.
   특성 A: 범위 [0, 5]   -> 그대로
   특성 B: 범위 [0, 3]   -> 값에 6을 더함 (A의 최대값+1)
   번들:  A 값 0-5, B 값 6-9

실제 효과 (NLP 등 희소 데이터):
  원래 특성 10,000개 -> EFB 후 300개 번들
  -> 히스토그램 계산 33배 빨라짐
```

# Level 2: LightGBM 고급 -- GOSS, 범주형, 하이퍼파라미터

## GOSS (Gradient-based One-Side Sampling) 전체 알고리즘

이전 글에서 GOSS의 아이디어를 간단히 봤다. 이제 정확한 알고리즘을 본다.

핵심 통찰: **그래디언트의 크기가 큰 샘플이 정보 이득에 더 많이 기여한다.**

```
왜 그래디언트 크기가 중요한가?

그래디언트 = 손실 함수의 1차 미분 = "현재 예측이 정답에서 얼마나 벗어나는가"

그래디언트가 큰 샘플: 현재 모델이 크게 틀리고 있다
  -> 이 샘플에서 배울 것이 많다
  -> 분할 시 정보 이득에 큰 영향을 미친다

그래디언트가 작은 샘플: 현재 모델이 거의 맞추고 있다
  -> 이미 잘 학습된 상태
  -> 분할 시 정보 이득에 작은 영향
  -> 일부만 써도 충분하다

수학적으로:
  분할의 정보 이득 = f(그래디언트 합, 헤시안 합)
  그래디언트가 큰 소수의 샘플이 합에서 지배적인 비중을 차지.
```

GOSS 알고리즘 (단계별):

```
입력: 학습 데이터 n개, 파라미터 a (상위 비율), b (하위 샘플링 비율)

1. 모든 샘플의 그래디언트 절대값을 계산한다.
   |g₁|, |g₂|, ..., |gₙ|

2. 그래디언트 절대값 기준으로 내림차순 정렬한다.

3. 상위 a% 샘플을 전부 선택한다. -> 집합 A
   이들은 현재 모델이 가장 못 맞추는 샘플이다.

4. 나머지 (1-a)% 중에서 b%를 랜덤 샘플링한다. -> 집합 B
   이들은 이미 잘 맞추고 있는 샘플의 대표 표본이다.

5. 집합 B의 샘플에 가중치 (1-a)/b 를 곱한다.
   -> 소수의 샘플로 다수를 대표하므로 가중치를 높여서 보정.

6. 집합 A (가중치 1) + 집합 B (가중치 (1-a)/b) 로 트리를 학습한다.
```

구체적인 예시:

```
데이터: 1,000,000건 (100만 건)
a = 20%, b = 10%

Step 1: 100만 개의 그래디언트 계산

Step 2: 그래디언트 절대값으로 정렬

Step 3: 상위 20% = 200,000건 선택 (집합 A)
  -> 이들의 그래디언트: |g| > 0.5 (예시)
  -> 가중치: 1.0 (변경 없음)

Step 4: 나머지 80% = 800,000건 중 10% = 80,000건 랜덤 추출 (집합 B)
  -> 이들의 그래디언트: |g| < 0.5 (예시)
  -> 가중치: (1 - 0.2) / 0.1 = 8.0

Step 5: 학습에 사용하는 데이터:
  A: 200,000건 (가중치 1.0)
  B:  80,000건 (가중치 8.0)
  합계: 280,000건 -> 원래의 28%만 사용!

효과:
  학습 시간: 약 3.5배 단축 (280K/1000K)
  성능: 가중치 보정 덕분에 거의 동일
  -> 실제 벤치마크에서 정확도 차이 < 0.1% (대부분의 경우)
```

가중치 보정이 왜 필요한가:

```
정보 이득을 계산할 때 그래디언트의 합을 쓴다.

원래 (전체 데이터):
  G_left = Σ gᵢ (왼쪽 노드에 속한 모든 샘플의 그래디언트 합)

GOSS (일부 데이터):
  G_left = Σ gᵢ (A에서 왼쪽에 속한 것)
         + (1-a)/b × Σ gⱼ (B에서 왼쪽에 속한 것)

보정 없이 B를 그대로 쓰면:
  작은 그래디언트 샘플들의 기여가 과소 반영.
  -> 정보 이득 추정이 편향됨.

(1-a)/b 가중치를 곱하면:
  B의 80,000건이 원래 800,000건을 대표.
  각각에 8배 가중치 -> 합계가 원래와 비슷해짐.
  -> 정보 이득의 불편 추정량(unbiased estimator)에 근사.
```

## LightGBM의 범주형 특성 처리

LightGBM은 범주형 특성을 원-핫 인코딩 없이 직접 처리할 수 있다. 내부적으로 **Fisher의 최적 분할 알고리즘**을 사용한다.

```
문제: "도시" 특성에 값이 4개: 서울, 부산, 대구, 인천

가능한 분할: 2^(4-1) - 1 = 7가지
  {서울} vs {부산, 대구, 인천}
  {부산} vs {서울, 대구, 인천}
  {서울, 부산} vs {대구, 인천}
  ...

카테고리가 k개면 분할 후보가 2^(k-1) - 1개.
k=100이면 약 10^29가지 -> 전수 탐색 불가능!

Fisher의 알고리즘 (이진 분류/회귀):
  1. 각 카테고리의 평균 타겟값(또는 그래디언트 합/헤시안 합)을 계산
     서울: 0.72, 부산: 0.65, 인천: 0.58, 대구: 0.42

  2. 이 값으로 카테고리를 정렬
     대구(0.42), 인천(0.58), 부산(0.65), 서울(0.72)

  3. 이제 연속형처럼 취급: 정렬된 순서에서 분할점만 탐색
     {대구} vs {인천, 부산, 서울}
     {대구, 인천} vs {부산, 서울}
     {대구, 인천, 부산} vs {서울}

  -> 분할 후보가 k-1개로 줄어든다!
  -> k=100이어도 99개만 시도하면 된다.

증명: Fisher(1958)가 이진 분류에서 이 방법이 최적 분할을
      보장한다는 것을 증명했다.
```

# Level 3: CatBoost -- 범주형 특성의 왕

Yandex(러시아 검색 엔진)가 2017년에 발표한 그래디언트 부스팅 프레임워크다. 이름부터 **Cat**egorical **Boost**ing이다. 핵심 아이디어: 범주형 특성을 원-핫 인코딩 없이 직접 처리하되, **데이터 누출(leakage)을 원천 차단**한다.

## 나이브 타겟 인코딩의 문제: 데이터 누출

범주형 특성을 숫자로 바꾸는 가장 직관적인 방법은 **타겟 인코딩(Target Encoding)**이다. 각 카테고리를 해당 카테고리의 평균 타겟값으로 대체한다.

```
원본 데이터:
  도시    연봉     구매여부(타겟)
  서울    5000     1
  부산    3000     0
  서울    4500     1
  대구    3500     0
  부산    3200     1
  서울    6000     1

나이브 타겟 인코딩:
  서울의 평균 타겟: (1+1+1)/3 = 1.0
  부산의 평균 타겟: (0+1)/2 = 0.5
  대구의 평균 타겟: 0/1 = 0.0

  도시_encoded  연봉    구매여부
  1.0          5000    1
  0.5          3000    0
  1.0          4500    1
  0.0          3500    0
  0.5          3200    1
  1.0          6000    1

문제: "서울"의 인코딩 값 1.0을 계산할 때
      서울 샘플 자신의 타겟값(1, 1, 1)을 사용했다!
      -> 타겟 정보가 특성에 직접 유입 = 데이터 누출
      -> 학습 시 타겟을 이미 알고 있는 셈 = 과적합
```

데이터 누출이 심각한 경우:

```
카테고리에 샘플이 1개뿐이면:
  "제주": 샘플 1개, 타겟 = 1
  -> 인코딩 값 = 1.0
  -> 모델: "제주 = 1.0 -> 타겟 1" (완벽히 외움)
  -> 테스트에서 "제주" 가 나오면? 무조건 1로 예측 (틀릴 수 있음)

실제로 나이브 타겟 인코딩을 쓰면:
  학습 AUC: 0.99
  테스트 AUC: 0.75
  -> 극심한 과적합
```

## Ordered Target Statistics: CatBoost의 해결책

CatBoost는 각 샘플의 인코딩에 **그 샘플 자신의 타겟값을 사용하지 않는다.** 랜덤 순열(permutation)에서 해당 샘플보다 **앞에 있는 샘플들의 타겟값만** 사용한다.

5개 샘플로 단계별 워크스루:

```
원본 데이터 (번호는 편의상):
  #   도시   타겟
  1   서울    1
  2   부산    0
  3   서울    1
  4   서울    0
  5   부산    1

랜덤 순열 sigma: [3, 1, 5, 2, 4]
(데이터를 이 순서로 섞었다고 생각하자)

순열 순서대로 인코딩 계산 (prior = 0.5, prior_count = 1):

공식: TS(i) = (앞선 같은 카테고리 샘플의 타겟 합 + prior * prior_count)
             / (앞선 같은 카테고리 샘플 수 + prior_count)

--- 순서 1: 샘플 #3 (서울, 타겟=1) ---
  앞에 서울이 0개. 참고할 것이 없다.
  TS(3) = (0 + 0.5 * 1) / (0 + 1) = 0.5
  (사전 확률만 사용)

--- 순서 2: 샘플 #1 (서울, 타겟=1) ---
  앞에 서울이 1개: 샘플 #3 (타겟=1)
  TS(1) = (1 + 0.5) / (1 + 1) = 0.75

--- 순서 3: 샘플 #5 (부산, 타겟=1) ---
  앞에 부산이 0개.
  TS(5) = (0 + 0.5) / (0 + 1) = 0.5

--- 순서 4: 샘플 #2 (부산, 타겟=0) ---
  앞에 부산이 1개: 샘플 #5 (타겟=1)
  TS(2) = (1 + 0.5) / (1 + 1) = 0.75

--- 순서 5: 샘플 #4 (서울, 타겟=0) ---
  앞에 서울이 2개: 샘플 #3 (타겟=1), 샘플 #1 (타겟=1)
  TS(4) = (1+1 + 0.5) / (2 + 1) = 2.5/3 = 0.833

결과:
  #   도시   타겟   인코딩
  1   서울    1     0.75     <- 자기 타겟(1) 안 씀, #3만 참고
  2   부산    0     0.75     <- 자기 타겟(0) 안 씀, #5만 참고
  3   서울    1     0.50     <- 앞에 아무도 없어서 prior 사용
  4   서울    0     0.833    <- 자기 타겟(0) 안 씀, #3,#1만 참고
  5   부산    1     0.50     <- 앞에 아무도 없어서 prior 사용

핵심: 어떤 샘플도 자기 자신의 타겟값을 보지 못했다!
     -> 데이터 누출 없음.
```

prior와 prior_count의 역할:

```
prior (보통 데이터 전체의 평균 타겟값):
  앞에 참고할 샘플이 없을 때 사용하는 기본값.
  베이즈의 사전 확률과 같은 역할.

prior_count (보통 1):
  prior에 부여하는 가상의 샘플 수.
  클수록 prior의 영향이 강하다 (보수적).
  작을수록 관찰 데이터의 영향이 강하다 (공격적).

  prior_count = 1:  1개 가상 샘플 -> 빠르게 데이터에 적응
  prior_count = 10: 10개 가상 샘플 -> 천천히 데이터에 적응
  -> 카테고리 내 샘플 수가 적을 때 과적합 방지
```

## 다중 순열로 안정성 확보

하나의 순열만 쓰면 인코딩이 순열에 따라 크게 달라진다. CatBoost는 **여러 개의 랜덤 순열**을 사용하여 이를 완화한다.

```
순열 1: [3, 1, 5, 2, 4]  -> 인코딩 세트 1
순열 2: [2, 4, 1, 5, 3]  -> 인코딩 세트 2
순열 3: [5, 3, 2, 1, 4]  -> 인코딩 세트 3
순열 4: [1, 2, 3, 4, 5]  -> 인코딩 세트 4

학습 과정:
  트리 1: 순열 1의 인코딩으로 학습
  트리 2: 순열 2의 인코딩으로 학습
  트리 3: 순열 3의 인코딩으로 학습
  ...

각 트리가 다른 인코딩을 보므로:
  -> 특정 순열에 의한 편향이 평균화된다
  -> 앙상블 효과와 결합되어 안정적인 성능
```

## Ordered Boosting: 예측 시프트 방지

CatBoost의 또 다른 혁신이다. 일반적인 그래디언트 부스팅에는 **예측 시프트(prediction shift)** 문제가 있다.

```
일반 그래디언트 부스팅의 문제:

트리 t를 학습할 때:
  잔차(그래디언트) = 실제값 - F_{t-1}(x)  (이전까지의 모델 예측)

여기서 F_{t-1}(x)는 학습 데이터로 만든 모델이다.
그런데 잔차를 계산하는 x도 같은 학습 데이터이다!

-> F_{t-1}이 학습 데이터에 과적합되어 있으면
   잔차가 실제보다 작게 나온다 (이미 외웠으니까).
-> 다음 트리가 "별로 틀린 게 없네"라고 판단.
-> 테스트 데이터에서는 잔차가 크게 나타남.
-> 학습 데이터와 테스트 데이터의 잔차 분포가 다름 = 예측 시프트.
```

Ordered Boosting의 해결:

```
아이디어: 샘플 i의 잔차를 계산할 때,
          샘플 i를 사용하지 않고 만든 모델로 계산한다.

순열 순서: [3, 1, 5, 2, 4]

샘플 #3의 잔차: 앞에 아무도 없으므로 초기 모델(상수)로 계산
샘플 #1의 잔차: #3만으로 학습한 모델로 계산
샘플 #5의 잔차: #3, #1로 학습한 모델로 계산
샘플 #2의 잔차: #3, #1, #5로 학습한 모델로 계산
샘플 #4의 잔차: #3, #1, #5, #2로 학습한 모델로 계산

-> 각 샘플의 잔차는 "자기 자신을 본 적 없는" 모델로부터 계산됨.
-> 예측 시프트가 원천 차단됨.

대가: n개 샘플에 대해 n개의 서로 다른 모델을 유지해야 한다.
     -> 메모리와 계산량이 증가.
     -> CatBoost는 이를 효율적으로 구현 (대칭 트리 활용).
```

## 대칭 트리 (Oblivious Decision Trees)

CatBoost가 사용하는 트리 구조다. 일반 트리와 근본적으로 다르다.

```
일반 트리 (비대칭):
  각 노드마다 다른 분할 조건을 사용할 수 있다.

         [나이 > 30?]
        /            \
  [소득 > 5000?]   [학력 = 대졸?]
    /      \          /        \
  Yes     No       [소득 > 3000?]  No
                    /         \
                  Yes         No

  깊이 3에서 서로 다른 분할 조건이 3종류.
  노드마다 독립적인 분할.

대칭 트리 (CatBoost):
  같은 깊이의 모든 노드가 동일한 분할 조건을 사용한다.

깊이 0:        [나이 > 30?]
              /            \
깊이 1:  [소득 > 5000?]   [소득 > 5000?]    <- 같은 조건!
          /      \          /        \
깊이 2: [학력?]  [학력?]  [학력?]   [학력?]   <- 같은 조건!
        / \      / \      / \       / \
       L1 L2   L3 L4    L5 L6     L7 L8

  깊이 3에서 분할 조건이 딱 3개.
  각 리프는 3개의 이진 조건의 조합으로 결정된다.

  L1: 나이>30 AND 소득>5000 AND 학력=대졸
  L2: 나이>30 AND 소득>5000 AND 학력!=대졸
  L3: 나이>30 AND 소득<=5000 AND 학력=대졸
  ...
```

대칭 트리의 장점:

```
1. 예측 속도가 빠르다.
   일반 트리: 루트에서 리프까지 조건을 하나씩 따라간다.
             각 노드에서 분기 -> O(깊이) 비교 (조건이 다 다르므로)

   대칭 트리: 각 깊이의 조건 결과를 비트로 표현.
             깊이 0 조건: 1 (나이>30이면)
             깊이 1 조건: 0 (소득<=5000이면)
             깊이 2 조건: 1 (학력=대졸이면)
             -> 비트열 101 = 인덱스 5 -> L5
             -> 배열 접근 한 번으로 리프 값을 찾음!
             -> 캐시 친화적, SIMD 연산에도 유리

   벤치마크 (트리 1000개, 깊이 6):
     일반 트리 예측:  0.5ms / 샘플
     대칭 트리 예측:  0.1ms / 샘플  (5배 빠름)

2. 자연스러운 정규화 효과.
   같은 깊이에서 같은 조건을 써야 하므로:
   -> 특정 경로만 과도하게 세밀해지는 것을 방지
   -> 트리가 데이터의 전역적 패턴에 집중
   -> 비대칭 트리보다 과적합에 강함

3. 구조가 단순하여 병렬화가 쉽다.
   각 깊이에서 하나의 분할 조건만 찾으면 된다.
   -> GPU 연산에 특히 유리
```

## CatBoost 핵심 하이퍼파라미터

```
[트리 구조]

depth (기본값: 6)
  대칭 트리의 깊이. 리프 수 = 2^depth.
  ┌─────────┬─────────┬──────────────┐
  │ depth   │ 리프 수  │ 적용 상황     │
  ├─────────┼─────────┼──────────────┤
  │ 4-5     │ 16-32   │ 데이터 적을 때│
  │ 6       │ 64      │ 일반적 (기본) │
  │ 8-10    │ 256-1024│ 데이터 많을 때│
  └─────────┴─────────┴──────────────┘

l2_leaf_reg (기본값: 3)
  리프 값에 대한 L2 정규화 계수.
  권장: 1-10

[학습 파라미터]

learning_rate (기본값: 자동)
  CatBoost는 iterations에 따라 자동으로 결정한다.
  수동 설정 시: 0.01-0.1

iterations (기본값: 1000)
  트리의 총 개수.
  early_stopping_rounds와 함께 큰 값으로 설정.

[범주형 특성]

cat_features
  범주형 특성의 인덱스나 이름 리스트.
  지정하면 Ordered Target Statistics가 자동 적용.

one_hot_max_size (기본값: 2)
  카테고리 수가 이 값 이하이면 원-핫 인코딩 사용.
  그 이상이면 Target Statistics 사용.

[과적합 방지]

random_strength (기본값: 1)
  분할 점수에 추가하는 랜덤 노이즈의 크기.
  클수록 더 랜덤한 분할 -> 과적합 방지.

bagging_temperature (기본값: 1)
  베이지안 부트스트랩의 온도 파라미터.
  0이면 부트스트랩 없음, 클수록 강한 부트스트랩.
```

```python
from catboost import CatBoostClassifier

model = CatBoostClassifier(
    iterations=5000,
    depth=6,
    learning_rate=0.05,
    l2_leaf_reg=3,
    random_strength=1,
    bagging_temperature=1,
    cat_features=['도시', '학력', '직업'],
    eval_metric='AUC',
    early_stopping_rounds=50,
    verbose=100
)

model.fit(
    X_train, y_train,
    eval_set=(X_val, y_val)
)
```

# XGBoost vs LightGBM vs CatBoost 상세 비교

## 아키텍처 비교표

```
                     XGBoost            LightGBM           CatBoost
─────────────────────────────────────────────────────────────────────────
트리 성장 방식        Level-wise         Leaf-wise          Level-wise (대칭)
분할 탐색            히스토그램          히스토그램           히스토그램
범주형 특성          원-핫 인코딩 필요    직접 처리 가능       직접 처리 (핵심 기능)
2차 근사             O                  O                  O
정규화              L1/L2 + gamma       L1/L2 + gamma      L2 + random_strength
샘플링              column/row sampling  GOSS + EFB         Ordered Boosting
결측값 처리          자동 (학습)          자동 (학습)         자동 (학습)
GPU 지원            O                  O                  O (더 빠름)
대칭 트리            X                  X                  O (기본)
예측 시프트 방지      X                  X                  O
멀티클래스          softmax             softmax             softmax
순위 학습            O                  O (LambdaRank)     O (YetiRank)
```

## 언제 무엇을 쓸 것인가

```
[데이터 크기 기준]

소규모 (1만건 이하):
  1순위: CatBoost
    -> Ordered Boosting이 소규모에서 과적합을 잘 막는다.
    -> 기본 설정만으로도 좋은 성능.
  2순위: XGBoost
    -> 안정적이고 검증된 성능.

중규모 (1만-100만건):
  1순위: LightGBM
    -> 속도와 성능의 균형.
  2순위: XGBoost
    -> 약간 느리지만 비슷한 성능.

대규모 (100만건 이상):
  1순위: LightGBM
    -> GOSS + EFB로 속도 압도적.
    -> 메모리 효율도 좋다.
  2순위: CatBoost (GPU 사용 시)
    -> GPU 구현이 잘 되어 있어서 대규모에서도 경쟁력.

[특성 유형 기준]

범주형 특성이 많을 때:
  1순위: CatBoost
    -> 전처리 없이 직접 처리. 데이터 누출 방지.
    -> 카디널리티 높은 범주형에서 특히 강함.
  2순위: LightGBM
    -> Fisher 알고리즘으로 범주형 처리 가능.
    -> CatBoost보다는 단순한 처리.

수치형 특성만 있을 때:
  1순위: LightGBM (속도 우선) 또는 XGBoost (안정성 우선)
  -> 범주형 장점이 없으므로 CatBoost의 이점이 줄어든다.

희소(sparse) 데이터:
  1순위: LightGBM
    -> EFB가 희소 특성을 번들링하여 차원을 줄인다.
  2순위: XGBoost
    -> 희소 데이터에 최적화된 알고리즘 있음.

[요구사항 기준]

빠른 프로토타이핑:
  LightGBM -> 학습이 가장 빠르다.

최고 성능(대회):
  셋 다 학습 후 앙상블 -> 0.001의 차이를 쫓을 때.

프로덕션 배포:
  CatBoost -> 대칭 트리 덕분에 예측(inference)이 가장 빠르다.
  LightGBM -> 모델 크기가 작고 예측도 빠르다.

해석 가능성:
  XGBoost -> 가장 오래되어 SHAP 등 도구와의 호환이 좋다.
  실제로 셋 다 SHAP을 지원한다.
```

## Kaggle 대회 분석

```
[2015-2017: XGBoost 전성기]
  Kaggle의 "정형 데이터" 대회 승자의 ~70%가 XGBoost 사용.
  "XGBoost를 쓰지 않으면 입상할 수 없다"는 말이 돌 정도.
  대표 사례: Higgs Boson, Otto Group, Santander

[2017-2019: LightGBM의 부상]
  속도 장점으로 빠르게 XGBoost를 대체.
  대규모 대회에서 LightGBM이 압도적:
  -> 탐색할 하이퍼파라미터 조합이 더 많아짐 (같은 시간에).
  대표 사례: Avito, Elo Merchant, IEEE Fraud Detection

[2019-현재: 삼국지 + 앙상블]
  단일 프레임워크 승리보다 앙상블이 일반적.
  전형적인 우승 솔루션:
    LightGBM (3-5 fold) + CatBoost (3-5 fold) + XGBoost (3-5 fold)
    -> 3개 프레임워크의 예측을 가중 평균.

  CatBoost가 특히 강한 대회 유형:
    -> 범주형 특성이 많은 추천/광고 대회
    -> 데이터가 작은 대회 (과적합 방지 덕분)

  LightGBM이 특히 강한 대회 유형:
    -> 대규모 데이터 (100만건 이상)
    -> 빠른 실험 사이클이 중요한 경우

현재 정형 데이터 대회의 "표준 파이프라인":
  1. LightGBM으로 빠르게 베이스라인 + 특성 엔지니어링
  2. CatBoost로 범주형 특성 실험
  3. XGBoost를 다양성을 위해 추가
  4. 3개를 스태킹 또는 가중 평균으로 앙상블
  5. Optuna로 하이퍼파라미터 자동 탐색
```

## 이 글에서 가져가야 할 것

```
Level 1:
  LightGBM의 Leaf-wise 성장은 이득이 가장 큰 리프만 분할한다.
  같은 리프 수에서 Level-wise보다 낮은 손실을 달성한다.
  히스토그램(256 bins)으로 분할점 탐색을 상수 시간으로 만든다.
  EFB로 상호 배타적인 희소 특성을 번들링하여 차원을 줄인다.

Level 2:
  GOSS는 그래디언트 상위 a%를 전부, 나머지에서 b%를 샘플링한다.
  가중치 (1-a)/b로 보정하여 통계적 편향을 막는다.
  핵심 파라미터는 num_leaves, learning_rate, feature_fraction.

Level 3:
  CatBoost는 Ordered Target Statistics로 데이터 누출을 차단한다.
  대칭 트리는 같은 깊이에서 같은 조건을 쓴다 -> 빠른 예측, 자연 정규화.
  Ordered Boosting은 예측 시프트를 방지한다.
  실전에서는 LightGBM + CatBoost + XGBoost 앙상블이 표준.
```
