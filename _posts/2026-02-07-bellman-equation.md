## 시작: "가치"란 무엇인가

강화학습에서 에이전트는 매 순간 선택을 한다. 어떤 선택이 좋은지 판단하려면 **"이 상태가 얼마나 좋은가"**를 수치화해야 한다. 이것이 **가치(value)**다.

```
미로 찾기:

  [시작] → [A] → [B] → [출구 +10]
                   ↘
                   [C] → [막다른길 0]

상태 B의 가치: 한 걸음만 가면 출구 → 가치가 높다
상태 C의 가치: 막다른길 → 가치가 낮다
상태 A의 가치: B로 갈 수도, C로 갈 수도 → 정책에 따라 다르다
```

가치를 정확히 정의하면: **"이 상태에서 시작해서 앞으로 받을 보상의 총합의 기댓값"**이다.

---

## 누적 보상과 할인

### 왜 미래 보상을 할인하는가

지금 받는 100원과 1년 뒤 받는 100원은 다르다. 미래는 불확실하고, 지금 받으면 바로 쓸 수 있다. 강화학습에서도 미래 보상에 **할인율 γ (감마, 0~1)**를 곱한다.

```
γ = 0.9일 때:

시점 t의 보상 r_t 의 현재 가치:
  t=0: r₀ × γ⁰ = r₀ × 1.0     ← 지금 받는 건 100%
  t=1: r₁ × γ¹ = r₁ × 0.9     ← 한 스텝 뒤는 90%
  t=2: r₂ × γ² = r₂ × 0.81    ← 두 스텝 뒤는 81%
  t=3: r₃ × γ³ = r₃ × 0.729
  ...
  t=10: r₁₀ × γ¹⁰ = r₁₀ × 0.349  ← 10스텝 뒤는 35%

먼 미래의 보상은 현재 가치로 환산하면 작아진다.
```

**누적 할인 보상 (Return):**

$$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

### 구체적으로 계산해보자

```
에이전트가 4스텝 동안 보상 [3, 1, 4, 2]를 받았다고 하자. γ = 0.9

t=0에서의 Return:
  G₀ = 3 + 0.9×1 + 0.9²×4 + 0.9³×2
     = 3 + 0.9 + 3.24 + 1.458
     = 8.598

t=1에서의 Return:
  G₁ = 1 + 0.9×4 + 0.9²×2
     = 1 + 3.6 + 1.62
     = 6.22

t=2에서의 Return:
  G₂ = 4 + 0.9×2
     = 4 + 1.8
     = 5.8
```

여기서 핵심적인 관계가 보인다:

```
G₀ = 3 + 0.9 × (1 + 0.9×4 + 0.9²×2)
   = 3 + 0.9 × G₁
   = r₁ + γ × G₁

G₁ = 1 + 0.9 × (4 + 0.9×2)
   = 1 + 0.9 × G₂
   = r₂ + γ × G₂
```

일반화하면:

$$G_t = r_{t+1} + \gamma \cdot G_{t+1}$$

**현재의 Return = 즉각 보상 + 할인된 미래의 Return**

이것이 벨만 방정식의 출발점이다. G를 기댓값으로 바꾸면 가치 함수가 된다.

## 상태 가치 함수 V(s)

### 정의

상태 s에서 시작하여 정책 π를 따를 때 기대되는 Return:

$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$$

"이 상태가 평균적으로 얼마나 좋은가"를 하나의 숫자로 나타낸다.

### 벨만 방정식 유도 — 왜 이 식이 되는가

$$G_t = r_{t+1} + \gamma \cdot G_{t+1}$$

양변에 기댓값을 씌우면:

$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid S_t = s]$$

$$= \mathbb{E}_\pi[r_{t+1} + \gamma \cdot G_{t+1} \mid S_t = s]$$

기댓값의 선형성으로 분리:

$$= \mathbb{E}_\pi[r_{t+1} \mid S_t = s] + \gamma \cdot \mathbb{E}_\pi[G_{t+1} \mid S_t = s]$$

두 번째 항에서 $$G_{t+1}$$의 기댓값은 다음 상태 $$S_{t+1}$$의 가치 함수와 같다:

$$= \mathbb{E}_\pi[r_{t+1} \mid S_t = s] + \gamma \cdot \mathbb{E}_\pi[V^\pi(S_{t+1}) \mid S_t = s]$$

합치면:

$$\boxed{V^\pi(s) = \mathbb{E}_\pi[r_{t+1} + \gamma \cdot V^\pi(S_{t+1}) \mid S_t = s]}$$

**현재 가치 = 즉각 보상의 기댓값 + 할인된 다음 상태 가치의 기댓값**

이것이 **벨만 기대 방정식(Bellman Expectation Equation)**이다.

### 왜 이게 강력한가

```
G_t를 직접 계산하려면:
  미래의 모든 보상을 끝까지 시뮬레이션해야 한다
  → 무한 시간이 걸릴 수 있다

벨만 방정식을 쓰면:
  V(s) = (즉각 보상) + γ × V(다음 상태)
  → 한 스텝 앞만 보면 된다!
  → 다음 상태의 V를 알면 현재 V를 구할 수 있다
  → 재귀적 구조

이것이 Dynamic Programming(동적 프로그래밍)의 핵심이다.
큰 문제를 작은 부분 문제로 나누어 해결한다.
```

### 그리드 월드 예제로 직접 계산

```
3×3 그리드. 출구에 도달하면 +10. 벽에 부딪히면 -1. 매 스텝 -0.1.
γ = 0.9. 정책: 모든 방향 균등 확률 (25%씩).

┌─────┬─────┬─────┐
│  A  │  B  │ 출구 │
│     │     │ +10  │
├─────┼─────┼─────┤
│  D  │  E  │  F  │
│     │     │     │
├─────┼─────┼─────┤
│  G  │  H  │  I  │
│     │     │     │
└─────┴─────┴─────┘
```

출구의 V는 알고 있다: V(출구) = 10 (도착하면 끝).

**B의 V를 구해보자:**

```
B에서 가능한 행동 (균등 정책, 각 25%):
  위:   벽에 부딪힘 → B에 그대로 (-0.1)
  아래: E로 이동 (-0.1)
  왼쪽: A로 이동 (-0.1)
  오른쪽: 출구 도착! (-0.1 + 10 = 9.9)

V(B) = 0.25 × [-0.1 + 0.9×V(B)]    ← 위 (제자리)
     + 0.25 × [-0.1 + 0.9×V(E)]    ← 아래
     + 0.25 × [-0.1 + 0.9×V(A)]    ← 왼쪽
     + 0.25 × [-0.1 + 0.9×V(출구)]  ← 오른쪽

V(B)가 양변에 다 등장한다!
다른 상태의 V도 마찬가지로 서로 연결되어 있다.
→ 연립방정식을 풀어야 한다 (또는 반복 계산으로 수렴)
```

**반복 계산 (Iterative Policy Evaluation):**

```
모든 V를 0으로 초기화하고, 벨만 방정식을 반복 적용:

반복 0: V(A)=0, V(B)=0, V(E)=0, ...

반복 1: V(B) = 0.25×[-0.1+0] + 0.25×[-0.1+0] + 0.25×[-0.1+0] + 0.25×[-0.1+0.9×10]
             = 0.25×(-0.1) × 3 + 0.25×(8.9)
             = -0.075 + 2.225
             = 2.15

반복 2: V(B)를 다시 계산 (이번에는 V(A), V(E) 등도 업데이트된 값 사용)
         → 더 정확한 값

...반복하면 수렴...

최종: V(B) ≈ 5.2  (출구에 가까우니 높다)
     V(G) ≈ 1.8  (출구에서 멀어서 낮다)
```

## 행동 가치 함수 Q(s, a)

### 정의

V(s)는 "이 상태가 얼마나 좋은가"지만, 에이전트가 실제로 필요한 것은 **"이 상태에서 이 행동을 하면 얼마나 좋은가"**다.

$$Q^\pi(s, a) = \mathbb{E}_\pi[G_t \mid S_t = s, A_t = a]$$

### V와 Q의 관계 — 왜 이렇게 연결되는가

**V에서 Q로:**

```
V(s) = "상태 s에서 정책 π를 따를 때의 가치"
     = "정책에 따라 행동을 고르고, 그 행동의 Q값을 기댓값으로"

V(s) = Σ_a π(a|s) × Q(s, a)
       ↑ 각 행동의 확률  ↑ 그 행동의 가치

예: 상태 B에서 정책이 (위25%, 아래25%, 왼25%, 오른25%)이면
V(B) = 0.25×Q(B,위) + 0.25×Q(B,아래) + 0.25×Q(B,왼) + 0.25×Q(B,오른)
```

**Q에서 V로:**

```
Q(s, a) = "상태 s에서 행동 a를 했을 때의 가치"
        = "즉각 보상 + 다음 상태의 V"

Q(s, a) = Σ_{s'} P(s'|s,a) × [R(s,a,s') + γ × V(s')]
          ↑ 다음 상태의 확률    ↑ 보상         ↑ 다음 상태의 가치

왜 합산(Σ)인가?
  같은 행동을 해도 환경이 확률적이면 여러 다음 상태가 가능하다.
  예: 미끄러운 바닥에서 "오른쪽"을 선택해도
      80% 확률로 오른쪽, 10% 위, 10% 아래로 갈 수 있다.
```

이 두 식을 서로에게 대입하면 V만의 벨만 방정식, Q만의 벨만 방정식이 나온다.

```
V → Q → V → Q → ... 재귀적으로 연결

V(s) = Σ_a π(a|s) × Σ_{s'} P(s'|s,a) × [R + γ × V(s')]

이것이 벨만 기대 방정식의 완전한 형태다.
```

## 최적 벨만 방정식 — 최고의 정책

### 정책을 개선하고 싶다

지금까지는 "정책 π가 주어졌을 때"의 가치를 구했다. 하지만 우리가 진짜 원하는 것은 **최적 정책(optimal policy)**이다.

```
균등 정책:   모든 방향 25%씩 → V(B) ≈ 5.2
좋은 정책:   오른쪽(출구) 100%  → V(B) ≈ 9.9 - 0.1 = 9.8
최적 정책:   모든 상태에서 최적 행동 → 모든 V가 최대화
```

### 최적 가치 함수

$$V^*(s) = \max_\pi V^\pi(s)$$

모든 가능한 정책 중에서 가장 높은 가치를 주는 정책의 V.

### 최적 벨만 방정식 유도

벨만 기대 방정식에서 "정책에 따라 행동을 고른다(Σ π(a|s))"를 "가장 좋은 행동을 고른다(max_a)"로 바꾸면 된다.

```
벨만 기대 방정식:
  V(s) = Σ_a π(a|s) × [즉각 보상 + γ × V(다음 상태)]
         ↑ 정책에 따라 가중 평균

최적 벨만 방정식:
  V*(s) = max_a [즉각 보상 + γ × V*(다음 상태)]
          ↑ 가장 좋은 행동을 선택
```

$$\boxed{V^*(s) = \max_a \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma V^*(s') \right]}$$

Q에 대해서도:

$$\boxed{Q^*(s, a) = \sum_{s'} P(s'|s,a) \left[ R(s,a,s') + \gamma \max_{a'} Q^*(s', a') \right]}$$

### 그리드 월드에서 최적 정책 계산

```
B에서 최적 행동:
  Q*(B, 위)    = -0.1 + 0.9 × V*(B)    ← 제자리
  Q*(B, 아래)  = -0.1 + 0.9 × V*(E)    ← E로
  Q*(B, 왼쪽)  = -0.1 + 0.9 × V*(A)    ← A로
  Q*(B, 오른쪽) = -0.1 + 0.9 × V*(출구) = -0.1 + 9 = 8.9  ← 출구!

V*(B) = max(Q*(B, 위), Q*(B, 아래), Q*(B, 왼), Q*(B, 오른))
      = Q*(B, 오른쪽)  (출구로 가는 게 가장 좋으니까)
      = 8.9

최적 정책: B에서는 항상 오른쪽으로!
```

## 벨만 방정식이 쓰이는 알고리즘

벨만 방정식 자체는 "관계식"이다. 이것을 **어떻게 풀 것인가**에 따라 알고리즘이 나뉜다.

```
환경을 완전히 알 때 (P(s'|s,a)를 안다):
  → Dynamic Programming
  → Policy Iteration: 평가(V 계산) → 개선(π 업데이트) 반복
  → Value Iteration: 최적 벨만 방정식을 반복 적용

환경을 모를 때 (P를 모른다, 경험으로만 학습):
  → Monte Carlo: 에피소드 끝까지 실행 → G_t를 직접 계산 → V를 평균
  → TD Learning: 한 스텝 후 벨만 방정식으로 추정
    V(s) ← V(s) + α[r + γV(s') - V(s)]
           ↑ 학습률    ↑ 벨만 목표(target)   ↑ 현재 추정
  → Q-Learning: Q에 대해 최적 벨만 방정식 적용
    Q(s,a) ← Q(s,a) + α[r + γ max_a' Q(s',a') - Q(s,a)]
```

### Q-Learning이 벨만 방정식에서 나오는 과정

```
최적 벨만 방정식:
  Q*(s,a) = E[r + γ max_{a'} Q*(s',a')]

현실에서는 E(기댓값)를 정확히 모르므로, 한 번의 경험으로 근사:
  target = r + γ max_{a'} Q(s',a')     ← 한 번의 샘플

현재 추정과 target의 차이(TD error):
  δ = target - Q(s,a) = [r + γ max_{a'} Q(s',a')] - Q(s,a)

Q를 target 쪽으로 조금 이동:
  Q(s,a) ← Q(s,a) + α × δ

이것이 Q-Learning 업데이트 규칙이다.
벨만 방정식의 기댓값을 샘플 하나로 근사한 것.
경험이 쌓이면 평균적으로 진짜 Q*에 수렴한다.
```

## 정리

```
출발점:
  G_t = r_{t+1} + γ G_{t+1}
  "현재의 Return = 즉각 보상 + 할인된 미래의 Return"

기댓값을 씌우면:
  V(s) = E[r + γ V(s')]
  "현재 가치 = 즉각 보상 기댓값 + 할인된 다음 상태 가치 기댓값"

V와 Q의 관계:
  V(s) = Σ π(a|s) × Q(s,a)    (정책으로 가중 평균)
  Q(s,a) = Σ P(s'|s,a) × [R + γ V(s')]  (환경의 확률로 가중 평균)

최적 정책:
  V*(s) = max_a [R + γ V*(s')]   (가장 좋은 행동 선택)
  Q*(s,a) = R + γ max_{a'} Q*(s',a')

Q-Learning:
  Q(s,a) ← Q(s,a) + α[r + γ max Q(s',a') - Q(s,a)]
  = 최적 벨만 방정식의 샘플 기반 근사
```
