## 알고리즘

버블 정렬은 배열을 반복 순회하면서 **인접한 두 원소를 비교**하고, 순서가 잘못되어 있으면 교환한다. 한 번의 순회(pass)가 끝나면 가장 큰 원소가 배열의 끝으로 "떠오른다(bubble up)". 이 과정을 정렬이 완료될 때까지 반복한다.

1. 배열의 첫 번째 원소부터 시작하여 인접한 원소와 비교한다.
2. 앞의 원소가 뒤의 원소보다 크면 교환한다.
3. 배열 끝까지 반복하면 1 pass가 완료된다. 이때 가장 큰 원소가 맨 뒤에 위치한다.
4. 정렬되지 않은 범위에 대해 위 과정을 반복한다.
5. 한 pass에서 교환이 한 번도 일어나지 않으면 정렬이 완료된 것이므로 조기 종료한다.

---

## 구현

```java
void bubbleSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        boolean swapped = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break; // 조기 종료
    }
}
```

`swapped` 플래그로 조기 종료를 구현한다. 한 pass에서 교환이 없었다면 이미 정렬된 상태이므로 더 반복할 필요가 없다.

---

## 시간 복잡도

| 경우 | 복잡도 | 설명 |
|------|--------|------|
| 최선 | O(n) | 이미 정렬된 배열 -- 첫 pass에서 교환 없이 종료 |
| 평균 | O(n^2) | 비교 횟수: n(n-1)/2 |
| 최악 | O(n^2) | 역순 정렬된 배열 |

---

## 공간 복잡도

O(1) -- 추가 배열 없이 주어진 배열 안에서 교환만 수행하는 **제자리 정렬(in-place)**이다.

---

## 안정 정렬 (Stable Sort)

버블 정렬은 **안정 정렬**이다. 같은 값을 가진 원소가 있을 때, 원래의 상대적 순서가 정렬 후에도 유지된다. 인접한 원소를 비교할 때 `>` 조건(같으면 교환하지 않음)을 사용하기 때문이다.

---

## 실무에서의 위치

버블 정렬은 거의 모든 상황에서 퀵 정렬, 병합 정렬, 팀소트 등에 비해 느리다. 실무에서 사용되는 경우는 드물고, 주로 **정렬 알고리즘의 기본 개념을 학습하는 교육 목적**으로 다뤄진다.

다만, 데이터가 거의 정렬된 상태에서 조기 종료를 활용하면 O(n)에 가까운 성능을 낼 수 있다는 점은 알아둘 만하다.
