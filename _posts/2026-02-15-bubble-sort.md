# Level 1: 동작 원리

## 핵심 아이디어

배열을 왼쪽에서 오른쪽으로 훑으면서, **인접한 두 원소를 비교**하고 순서가 잘못되어 있으면 교환한다. 한 번 훑으면 가장 큰 원소가 맨 뒤로 "떠오른다(bubble up)". 이것을 반복하면 정렬이 완료된다.

## 전체 과정을 따라가보자

배열: `[5, 3, 8, 1, 2]`

```
═══ Pass 1: 가장 큰 원소(8)를 맨 뒤로 보낸다 ═══

[5, 3, 8, 1, 2]   5 > 3? YES → 교환
 ↑  ↑
[3, 5, 8, 1, 2]   5 > 8? NO  → 그대로
    ↑  ↑
[3, 5, 8, 1, 2]   8 > 1? YES → 교환
       ↑  ↑
[3, 5, 1, 8, 2]   8 > 2? YES → 교환
          ↑  ↑
[3, 5, 1, 2, 8]   ← 8이 맨 뒤로 떠올랐다!
               ✓
비교 4회, 교환 3회


═══ Pass 2: 두 번째로 큰 원소(5)를 뒤에서 두 번째로 보낸다 ═══
(8은 이미 확정이므로 마지막 원소는 비교 안 함)

[3, 5, 1, 2, 8]   3 > 5? NO
 ↑  ↑
[3, 5, 1, 2, 8]   5 > 1? YES → 교환
    ↑  ↑
[3, 1, 5, 2, 8]   5 > 2? YES → 교환
       ↑  ↑
[3, 1, 2, 5, 8]   ← 5가 제 자리로!
            ✓  ✓
비교 3회, 교환 2회


═══ Pass 3 ═══

[3, 1, 2, 5, 8]   3 > 1? YES → 교환
 ↑  ↑
[1, 3, 2, 5, 8]   3 > 2? YES → 교환
    ↑  ↑
[1, 2, 3, 5, 8]   ← 3이 제 자리로!
         ✓  ✓  ✓
비교 2회, 교환 2회


═══ Pass 4 ═══

[1, 2, 3, 5, 8]   1 > 2? NO
 ↑  ↑
[1, 2, 3, 5, 8]   ← 교환 없음! 이미 정렬 완료.
비교 1회, 교환 0회 → 조기 종료!
```

## 왜 "버블"인가

큰 원소가 오른쪽으로 한 칸씩 이동하는 모습이 물속에서 거품이 떠오르는 것과 같다.

```
Pass 1에서 8의 이동:

[5, 3, 8, 1, 2]
          ↓
[5, 3, 8, 1, 2]    8은 아직 제자리
          ↓
[5, 3, 1, 8, 2]    8이 한 칸 오른쪽으로
             ↓
[5, 3, 1, 2, 8]    8이 또 한 칸 → 맨 끝 도착!

8이 거품처럼 "떠올랐다"
```

반대로 작은 원소는 왼쪽으로 이동한다. 하지만 **한 pass에 한 칸만** 왼쪽으로 갈 수 있다. 이것이 버블 정렬이 느린 근본적인 이유다 (뒤에서 자세히).

## 불변식 (Invariant)

알고리즘의 정확성을 보장하는 핵심 성질:

```
Pass k가 끝나면, 배열의 뒤쪽 k개 원소는 최종 위치에 확정된다.

Pass 1 후: [..................., 8]     ← 뒤 1개 확정
Pass 2 후: [................, 5, 8]     ← 뒤 2개 확정
Pass 3 후: [............., 3, 5, 8]     ← 뒤 3개 확정
Pass 4 후: [........., 2, 3, 5, 8]     ← 뒤 4개 확정

n-1번의 pass 후 모든 원소가 확정 → 정렬 완료
```

# Level 2: 구현과 최적화

## 기본 구현 (C)

```c
void bubble_sort(int *arr, int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - 1 - i; j++) {  /* 확정된 뒤쪽은 건너뜀 */
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}
```

`n - 1 - i`: pass i가 끝나면 뒤쪽 i개는 확정이므로 비교할 필요 없다.

## 최적화 1: 조기 종료 (Early Termination)

한 pass에서 교환이 한 번도 없었다면 이미 정렬된 상태다.

```c
void bubble_sort_optimized(int *arr, int n) {
    for (int i = 0; i < n - 1; i++) {
        int swapped = 0;
        for (int j = 0; j < n - 1 - i; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }
        if (!swapped) break;  /* 교환 없으면 정렬 완료 */
    }
}
```

**효과:**

```
이미 정렬된 배열 [1, 2, 3, 4, 5]:
  기본: 4 + 3 + 2 + 1 = 10회 비교
  조기 종료: 4회 비교 후 종료 (Pass 1에서 교환 없음)
  → O(n²) → O(n)으로 개선!

거의 정렬된 배열 [1, 2, 4, 3, 5]:
  기본: 10회 비교
  조기 종료: Pass 1 (4회 비교, 1회 교환) + Pass 2 (3회 비교, 0회 교환) = 7회 비교
```

## 최적화 2: 마지막 교환 위치 기록

조기 종료보다 더 정밀한 최적화. 마지막 교환이 일어난 위치 이후는 이미 정렬된 상태이므로 다음 pass의 범위를 줄일 수 있다.

```c
void bubble_sort_last_swap(int *arr, int n) {
    int last = n - 1;  /* 다음 pass의 비교 범위 */
    while (last > 0) {
        int new_last = 0;
        for (int j = 0; j < last; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                new_last = j;  /* 마지막 교환 위치 기록 */
            }
        }
        last = new_last;  /* 이 위치 이후는 정렬 완료 */
    }
}
```

```
예: [1, 2, 5, 4, 3, 6, 7, 8]

Pass 1:
  비교 범위: 인덱스 0~6
  교환: (5,4) at j=2, (5,3) at j=3
  마지막 교환: j=3
  → [1, 2, 4, 3, 5, 6, 7, 8]

Pass 2:
  비교 범위: 인덱스 0~3 (last=3)  ← 4~7은 건너뜀!
  교환: (4,3) at j=2
  마지막 교환: j=2
  → [1, 2, 3, 4, 5, 6, 7, 8]

Pass 3:
  비교 범위: 인덱스 0~2
  교환: 없음 → 종료

기본 버전이면 7 + 6 + 5 + ... = 28회 비교
최적화하면 7 + 4 + 3 = 14회 비교 → 50% 절약
```

## 비교 횟수와 교환 횟수를 정확히 세기

```
배열 크기 n = 5

═══ 최악 (역순 배열 [5, 4, 3, 2, 1]) ═══

Pass 1: 비교 4회, 교환 4회  → [4, 3, 2, 1, 5]
Pass 2: 비교 3회, 교환 3회  → [3, 2, 1, 4, 5]
Pass 3: 비교 2회, 교환 2회  → [2, 1, 3, 4, 5]
Pass 4: 비교 1회, 교환 1회  → [1, 2, 3, 4, 5]

총 비교: 4 + 3 + 2 + 1 = 10 = n(n-1)/2
총 교환: 4 + 3 + 2 + 1 = 10 = n(n-1)/2

역순일 때 비교와 교환이 모두 최대.


═══ 최선 (이미 정렬된 [1, 2, 3, 4, 5]) ═══

Pass 1: 비교 4회, 교환 0회 → 조기 종료!

총 비교: 4 = n - 1
총 교환: 0


═══ n에 따른 최악 비교 횟수 ═══

n       비교 횟수 n(n-1)/2      대략
────────────────────────────────────
5           10                 10
10          45                 50
100       4,950              5,000
1,000   499,500            500,000
10,000 49,995,000        50,000,000

n이 10배 → 비교가 100배. 이것이 O(n²)의 의미.
```

## 교환 횟수 = 역전의 수 (Inversion Count)

버블 정렬의 교환 횟수는 배열의 **역전(inversion)** 수와 정확히 같다.

역전이란: i < j인데 arr[i] > arr[j]인 쌍(i, j).

```
[3, 1, 2]의 역전:
  (3,1): 3 > 1  ← 역전
  (3,2): 3 > 2  ← 역전
  (1,2): 1 < 2  ← 역전 아님

역전 수 = 2

버블 정렬의 교환 횟수:
  [3, 1, 2] → [1, 3, 2] (교환 1)
  [1, 3, 2] → [1, 2, 3] (교환 2)

교환 2회 = 역전 2개. 정확히 일치!
```

**왜 일치하는가?** 버블 정렬의 교환은 인접한 두 원소의 역전을 하나씩 해소한다. 인접하지 않은 원소의 역전은 영향을 받지 않는다. 따라서 총 교환 횟수 = 총 역전 수.

```
역전 수의 최대값:
  모든 쌍이 역전 → C(n,2) = n(n-1)/2
  [5, 4, 3, 2, 1]: 역전 10개 → 교환 10회

역전 수의 최소값:
  이미 정렬 → 0개
  [1, 2, 3, 4, 5]: 역전 0개 → 교환 0회

역전 수의 평균값:
  무작위 배열에서 각 쌍이 역전일 확률 = 1/2
  평균 역전 수 = n(n-1)/4
```

# Level 3: 복잡도 증명과 본질적 한계

## 시간 복잡도 증명

**최악 O(n²):**

```
최악 입력: 역순 배열

Pass k에서의 비교 횟수: n - k

총 비교 횟수:
  Σ(k=1 to n-1) (n-k) = (n-1) + (n-2) + ... + 1 = n(n-1)/2

n(n-1)/2 = (n² - n)/2 = O(n²)  ∎
```

**최선 O(n):**

```
최선 입력: 이미 정렬된 배열 (조기 종료 사용 시)

Pass 1: n-1번 비교, 0번 교환 → swapped = false → 종료

총 비교 횟수: n - 1 = O(n)  ∎
```

**평균 O(n²):**

```
평균 교환 횟수 = 평균 역전 수 = n(n-1)/4 = O(n²)

각 교환은 O(1)이고, 비교는 교환보다 많거나 같으므로
평균 시간 복잡도도 O(n²)  ∎
```

## 인접 교환 기반 정렬의 하한

**버블 정렬이 느린 것은 구현의 문제가 아니라 "인접 교환"이라는 접근 자체의 한계다.**

```
정리: 인접 원소만 교환하는 정렬 알고리즘은 최악 Ω(n²)이다.

증명:
  최악의 경우 역전 수 = n(n-1)/2
  인접 교환 1회는 역전을 정확히 1개 해소
  → 최소 n(n-1)/2회 교환 필요
  → Ω(n²)  ∎

이것은 버블 정렬뿐 아니라 삽입 정렬, 칵테일 셰이커 정렬 등
"인접 교환" 기반의 모든 정렬에 적용된다.

퀵 정렬, 병합 정렬이 O(n log n)을 달성하는 이유:
  "멀리 떨어진 원소를 한 번에 교환"하기 때문이다.
  → 한 번의 교환으로 여러 역전을 동시에 해소
```

## "토끼와 거북이" 문제

```
배열: [2, 3, 4, 5, 1]

1이 맨 뒤에서 맨 앞으로 가야 한다.

Pass 1: [2, 3, 4, 1, 5]  ← 1이 한 칸 왼쪽으로
Pass 2: [2, 3, 1, 4, 5]  ← 1이 한 칸 왼쪽으로
Pass 3: [2, 1, 3, 4, 5]  ← 1이 한 칸 왼쪽으로
Pass 4: [1, 2, 3, 4, 5]  ← 1이 한 칸 왼쪽으로

4 pass 필요!
```

```
큰 원소 = "토끼" (rabbit):
  오른쪽으로 한 pass에 끝까지 갈 수 있다 (빠르게 이동)

작은 원소 = "거북이" (turtle):
  왼쪽으로 한 pass에 한 칸만 이동 (느리게 이동)

→ 거북이가 버블 정렬의 속도를 결정한다
→ 맨 뒤에 작은 값이 있으면 n-1 pass가 필요
```

**칵테일 셰이커 정렬(Cocktail Shaker Sort)**은 이 문제를 완화한다. 좌→우로 한 번 훑고, 다시 우→좌로 훑는다. 거북이도 한 pass에 여러 칸 이동할 수 있다. 하지만 최악은 여전히 O(n²)이다.

## 안정 정렬 (Stable Sort)

같은 값을 가진 원소가 있을 때, **원래의 상대적 순서가 유지**되면 안정 정렬이다.

```
입력: [(3,a), (1,b), (3,c), (2,d)]
      (값, 식별자) 형태

안정 정렬 결과:   [(1,b), (2,d), (3,a), (3,c)]
                                   ↑ a가 c보다 앞 (원래 순서 유지)

불안정 정렬 결과: [(1,b), (2,d), (3,c), (3,a)]  ← 가능
                                   ↑ c가 a보다 앞 (순서 바뀜)
```

**버블 정렬이 안정인 이유:**

```c
if (arr[j] > arr[j + 1])  /* > 를 사용 (같으면 교환하지 않음) */
```

같은 값이면 교환하지 않으므로 원래 순서가 보존된다. 만약 `>=`를 쓰면 불안정이 된다.

**안정성이 왜 중요한가:**

```
학생 성적 데이터:
  이름순으로 정렬된 상태에서 점수순으로 다시 정렬

  원본 (이름순): [(Alice,85), (Bob,90), (Charlie,85), (Dave,90)]

  안정 정렬 (점수순):
    [(Alice,85), (Charlie,85), (Bob,90), (Dave,90)]
    → 같은 점수 내에서 이름순 유지! 의도한 결과.

  불안정 정렬 (점수순):
    [(Charlie,85), (Alice,85), (Dave,90), (Bob,90)]  ← 가능
    → 같은 점수 내에서 순서가 뒤바뀔 수 있음
```

## O(n²) 정렬 3종 비교

```
                    버블 정렬        선택 정렬        삽입 정렬
──────────────────────────────────────────────────────────────
최선               O(n)            O(n²)           O(n)
평균               O(n²)           O(n²)           O(n²)
최악               O(n²)           O(n²)           O(n²)
교환 횟수 (최악)   O(n²)           O(n)            O(n²)
안정성             안정            불안정           안정
조기 종료          가능            불가능           --
거의 정렬된 데이터  매우 빠름(O(n)) 항상 같음       매우 빠름(O(n))
핵심 연산          인접 비교+교환   최솟값 탐색      삽입 위치 탐색

같은 O(n²)인데 성격이 다르다:
  버블: 큰 값을 뒤로 "밀어낸다" (오른쪽부터 확정)
  선택: 최솟값을 "뽑아서 앞에 놓는다" (왼쪽부터 확정)
  삽입: 새 카드를 "적절한 위치에 끼워 넣는다" (왼쪽이 항상 정렬 상태)
```

```
같은 [5, 3, 8, 1, 2]를 정렬할 때 비교 횟수:

버블 (조기 종료):   4 + 3 + 2 + 1 = 10회 (최악 기준)
선택:              4 + 3 + 2 + 1 = 10회 (항상)
삽입:              1 + 1 + 3 + 3 = 8회  (이 배열에서)

삽입 정렬이 평균적으로 약간 빠른 경우가 많다.
이미 정렬된 부분에서 이진 탐색을 활용하면 비교를 더 줄일 수 있다.

실측 (n=10,000, 무작위):
  버블:  ~250ms
  선택:  ~100ms  (교환이 적어서)
  삽입:  ~80ms   (평균 비교가 적어서)
  퀵:    ~2ms    (O(n log n))

→ O(n²)끼리도 상수 배 차이가 있지만,
  O(n log n)과의 차이는 100배 이상.
```

## 정리

```
Level 1:
  인접한 두 원소를 비교하고 교환한다.
  한 pass에 가장 큰 원소가 맨 뒤로 떠오른다.
  pass를 반복하면 뒤에서부터 정렬이 확정된다.

Level 2:
  조기 종료: 교환 없으면 종료 → 최선 O(n)
  마지막 교환 위치 기록 → 비교 범위 축소
  교환 횟수 = 역전(inversion) 수

Level 3:
  인접 교환 기반은 Ω(n²) 하한을 깰 수 없다.
  거북이 문제: 작은 값이 뒤에 있으면 느리다.
  안정 정렬: 같은 값의 원래 순서를 보존한다.
  O(n²) 3종 중 삽입 정렬이 실측에서 보통 가장 빠르다.
```
