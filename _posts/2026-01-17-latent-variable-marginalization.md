이 글은 두 가지 질문에 답한다:
1. **잠재 변수(latent variable)**: 관측할 수 없는 변수가 왜 필요하고 어떻게 다루는가
2. **주변화(marginalization)**: 변수를 "합산하여 제거"하는 것이 무엇이고 왜 중요한가

이 두 개념은 GMM, EM 알고리즘, VAE, 확산 모델 등 현대 ML의 핵심 도구다.

# 잠재 변수란

## 직관: 보이지 않는 원인

데이터를 관측했는데, 그 데이터를 만들어낸 **숨은 원인**이 있다고 생각하는 것이다.

```
관측: 학생 100명의 시험 점수

  점수 분포를 보니 봉우리가 2개다:

  빈도
  |     *
  |    * *         *
  |   *   *       * *
  |  *     *     *   *
  | *       *   *     *
  |*         * *       *
  └──────────────────────── 점수
     40~60       80~100

  왜 봉우리가 2개인가?
  → "열심히 공부한 그룹"과 "안 한 그룹"이 섞여 있기 때문!
```

"어느 그룹에 속하는가"는 시험 점수에 적혀있지 않다. 관측되지 않는다. 하지만 이 **숨은 그룹 변수**가 점수 분포를 설명한다. 이것이 잠재 변수다.

```
관측 변수 (observed): x = 시험 점수  ← 데이터에 있다
잠재 변수 (latent):   z = 어느 그룹  ← 데이터에 없다. 추론해야 한다.
```

## 잠재 변수가 쓰이는 곳

```
문제                     관측 변수(x)          잠재 변수(z)
──────────────────────────────────────────────────────────
클러스터링 (GMM)         데이터 포인트         어느 클러스터에 속하는가
토픽 모델링 (LDA)        문서의 단어들          문서의 토픽 분포
이미지 생성 (VAE)        이미지 픽셀           이미지의 의미적 특성 (스타일, 포즈 등)
음성 인식               음성 파형              발화된 단어 시퀀스
Hidden Markov Model     관측 시퀀스            숨은 상태 시퀀스
```

모든 경우의 패턴: **z를 알면 x를 설명하기 쉬운데, z를 직접 관측할 수 없다.**

# 주변화란

## 직관: 관심 없는 변수를 합산하여 제거한다

두 변수 x, z의 결합 확률 P(x, z)가 있을 때, z에 관심이 없으면 z의 모든 가능한 값에 대해 합산하여 제거할 수 있다.

$$P(x) = \sum_z P(x, z)$$

연속 변수라면 합산 대신 적분:

$$P(x) = \int P(x, z) \, dz$$

이것이 **주변화(marginalization)**다. "z를 적분(합산)해서 밀어내고 x만 남긴다."

## 구체적 예제: 주사위 2개

```
주사위 A와 B를 동시에 던진다.
x = A의 눈, z = B의 눈

결합 확률: P(x=3, z=5) = 1/36  (둘 다 독립이므로)

"A가 3일 확률"만 알고 싶다면?
B(z)의 모든 값을 합산:

P(x=3) = P(x=3, z=1) + P(x=3, z=2) + ... + P(x=3, z=6)
       = 1/36 + 1/36 + 1/36 + 1/36 + 1/36 + 1/36
       = 6/36
       = 1/6  ✓
```

이 예제는 너무 단순하지만, 원리는 동일하다. 복잡한 모델에서도 "z를 모두 합산하면 z 없이 x만의 확률을 구할 수 있다."

## 조건부 확률과 곱셈 규칙을 결합

결합 확률을 곱셈 규칙으로 분해한 뒤 주변화할 수 있다:

$$P(x) = \sum_z P(x, z) = \sum_z P(x \mid z) \cdot P(z)$$

```
P(x|z): z가 주어졌을 때 x의 확률 (조건부)
P(z):   z의 사전 확률

이 분해가 핵심이다:
  P(x|z)와 P(z)를 각각 모델링하면
  → 합산으로 P(x)를 구할 수 있다
```

이것은 [조건부 확률](/conditional-probability/)의 전확률 정리와 같은 식이다. 잠재 변수 z가 전확률 정리의 "조건"에 해당한다.

# 잠재 변수 + 주변화 = 생성 모델

## 생성 모델의 기본 구조

```
생성 과정 (데이터가 만들어지는 과정을 상상):

  1. 잠재 변수 z를 사전분포 P(z)에서 뽑는다
     → "이 이미지는 웃는 얼굴, 금발, 정면"이라는 속성이 결정됨

  2. z가 주어지면 관측 데이터 x를 P(x|z)에서 생성한다
     → 그 속성에 맞는 이미지 픽셀이 생성됨

전체 결합 확률:
  P(x, z) = P(x|z) · P(z)

관측 데이터의 확률 (주변화):
  P(x) = Σ_z P(x|z) · P(z)      ← 이산
  P(x) = ∫ P(x|z) · P(z) dz    ← 연속
```

이 P(x)를 **주변 우도(marginal likelihood)** 또는 **증거(evidence)**라고 부른다. [베이즈 정리](/bayes-theorem/)의 분모 P(D)가 바로 이것이다.

# 왜 어려운가: 적분이 풀리지 않는다

## 문제

```
P(x) = ∫ P(x|z) · P(z) dz

이론적으로는 이 적분을 풀면 끝이다.
현실에서는 거의 항상 풀리지 않는다.

왜?
  z의 차원이 높으면 (예: 128차원)
  → 128차원 공간에서의 적분
  → 해석적 해가 없다
  → 수치 적분도 차원의 저주로 불가능
```

이 문제를 해결하는 방법에 따라 알고리즘이 나뉜다:

```
적분을 풀 수 있는 특수한 경우:
  → 닫힌 형태(closed-form) 해가 존재
  → 예: 가우시안 혼합의 각 성분이 가우시안일 때

적분을 근사하는 방법:
  1. EM 알고리즘: z의 사후분포를 추정 → 기댓값으로 적분을 우회
  2. MCMC: 사후분포에서 샘플링하여 적분을 몬테카를로 근사
  3. 변분 추론 (VI): P(z|x)를 단순한 분포 Q(z)로 근사 → VAE의 핵심
```

# 예제 1: 가우시안 혼합 모델 (GMM)

가장 대표적인 잠재 변수 모델이다. "데이터가 K개의 가우시안 분포의 혼합으로 생성되었다"고 가정한다.

## 생성 과정

```
K=2인 경우 (두 클러스터):

1. 동전을 던진다: z = 1 (확률 π₁) 또는 z = 2 (확률 π₂)
   → "이 데이터 포인트는 어느 클러스터에서 왔는가"

2. z=k이면 가우시안 N(μₖ, σₖ²)에서 x를 뽑는다

결합 확률:
  P(x, z=k) = P(z=k) · P(x|z=k)
             = πₖ · N(x; μₖ, σₖ²)

주변 확률 (z를 합산하여 제거):
  P(x) = Σₖ πₖ · N(x; μₖ, σₖ²)
       = π₁ · N(x; μ₁, σ₁²) + π₂ · N(x; μ₂, σ₂²)
```

```
시각적으로:

  클러스터 1                  클러스터 2
  N(μ₁=50, σ₁=10)           N(μ₂=80, σ₂=8)
  π₁ = 0.4                  π₂ = 0.6

       ╱╲                          ╱╲
      ╱  ╲                        ╱  ╲
     ╱    ╲                      ╱    ╲
    ╱      ╲       +            ╱      ╲
───╱────────╲──────────────────╱────────╲───
  30    50    70              60    80   100

합산하면 (주변화):

       ╱╲              ╱╲
      ╱  ╲            ╱  ╲
     ╱    ╲          ╱    ╲
    ╱      ╲        ╱      ╲
───╱────────╲──────╱────────╲───
  30    50    70  60    80   100

→ 봉우리 2개짜리 분포. 관측 데이터의 분포와 일치!
```

## EM 알고리즘: z를 모르는데 어떻게 학습하는가

문제: μ, σ, π를 학습하고 싶은데, z(어느 클러스터에 속하는지)를 모른다. 하지만 z를 알려면 μ, σ, π가 필요하다. 닭과 달걀 문제.

```
EM 알고리즘: 번갈아가며 추정

E-step (Expectation):
  현재 파라미터(μ, σ, π)로 각 데이터 포인트의 z를 추정
  "이 포인트가 클러스터 k에 속할 확률"을 계산

  γ(k) = P(z=k|x) = πₖ · N(x; μₖ, σₖ²) / Σⱼ πⱼ · N(x; μⱼ, σⱼ²)
                      ↑ 베이즈 정리!

M-step (Maximization):
  추정된 z(γ)를 이용해 파라미터를 업데이트

  μₖ = (Σ γᵢ(k) · xᵢ) / (Σ γᵢ(k))    ← γ로 가중 평균
  π_k = (Σ γᵢ(k)) / N                   ← 각 클러스터의 비율

반복 → 수렴
```

```
수치 예제 (K=2, 데이터 4개):

데이터: [30, 45, 75, 90]

초기 추정: μ₁=40, σ₁=15, π₁=0.5
          μ₂=70, σ₂=15, π₂=0.5

═══ E-step ═══
x=30: P(z=1|x=30) = 0.5·N(30;40,15) / [0.5·N(30;40,15) + 0.5·N(30;70,15)]
                    ≈ 0.5·0.018 / (0.5·0.018 + 0.5·0.0001)
                    ≈ 0.99  → 거의 확실히 클러스터 1

x=75: P(z=1|x=75) ≈ 0.04  → 거의 확실히 클러스터 2

x=45: P(z=1|x=45) ≈ 0.82  → 클러스터 1일 가능성 높음

═══ M-step ═══
μ₁ = (0.99×30 + 0.82×45 + 0.04×75 + 0.01×90) / (0.99+0.82+0.04+0.01)
   ≈ 39.7

μ₂ = (0.01×30 + 0.18×45 + 0.96×75 + 0.99×90) / (0.01+0.18+0.96+0.99)
   ≈ 79.8

→ 반복하면 μ₁→35~40, μ₂→80~85로 수렴
```

# 예제 2: VAE — 잠재 변수의 딥러닝 응용

> Kingma & Welling, "Auto-Encoding Variational Bayes", ICLR, 2014

VAE는 잠재 변수 모델을 딥러닝으로 확장한 것이다.

## 구조

```
생성 모델 (디코더):
  z ~ P(z) = N(0, I)       ← 사전분포: 표준 가우시안
  x ~ P(x|z) = Decoder(z)  ← 디코더 신경망이 z에서 x를 생성

추론 모델 (인코더):
  z ~ Q(z|x) = Encoder(x)  ← 인코더 신경망이 x에서 z를 추론
                              진짜 사후분포 P(z|x)를 근사

P(z|x)를 직접 계산하려면:
  P(z|x) = P(x|z)·P(z) / P(x)    ← 베이즈 정리
                           ↑ P(x) = ∫ P(x|z)P(z)dz ← 이 적분이 안 풀림!

→ P(z|x)를 직접 구할 수 없으므로
→ Q(z|x)라는 근사 분포를 학습한다 (변분 추론)
```

## 손실 함수: ELBO

```
VAE의 손실 = -ELBO = 복원 오차 + KL 발산

Loss = E_Q[-log P(x|z)]  +  KL(Q(z|x) || P(z))
       ↑ 복원 오차              ↑ 사전분포와의 거리
       "z에서 x를 잘             "인코더가 추정한 z의 분포가
        복원하는가"               사전분포(가우시안)에서
                                 너무 벗어나지 않게"
```

```
복원 오차만 최소화하면?
  → z 공간이 엉망이 돼도 상관없이 외우기만 함
  → 오토인코더와 동일. 생성 불가.

KL만 최소화하면?
  → z가 항상 N(0,I)에 가까워짐. x를 무시.
  → 아무 의미 없는 z. 생성해도 쓰레기.

둘의 균형:
  → z 공간이 의미 있게 구조화되면서 + x도 잘 복원
  → z 공간에서 연속적으로 이동하면 이미지도 연속적으로 변화!
```

KL 발산에 대해서는 [KL Divergence](/kl-divergence/) 포스트에서 자세히 다룬다.

# 주변화가 등장하는 다른 곳들

```
1. 전확률 정리
   P(A) = Σ P(A|Bᵢ) · P(Bᵢ)
   → Bᵢ를 주변화하여 P(A)를 구한다
   → 조건부 확률 포스트에서 다룬 "공장 불량률" 예제가 이것

2. 베이즈 정리의 분모
   P(D) = Σ P(D|Hᵢ) · P(Hᵢ)
   → 모든 가설에 대해 주변화 = 증거(evidence)
   → 이것이 계산 불가능하면 → MCMC, 변분 추론으로 우회

3. Hidden Markov Model
   P(관측 시퀀스) = Σ_{모든 숨은 상태 시퀀스} P(관측|숨은) · P(숨은)
   → 숨은 상태를 주변화. Forward Algorithm이 이를 효율적으로 계산.

4. 확산 모델 (Diffusion Model)
   P(x₀) = ∫ P(x₀|x₁) P(x₁|x₂) ... P(x_{T-1}|x_T) P(x_T) dx₁...dx_T
   → 중간 노이즈 단계(x₁...x_T)가 잠재 변수
   → 이 적분을 ELBO로 근사 (VAE와 같은 원리!)
```

---

## 정리

```
잠재 변수 (z):
  관측되지 않지만 데이터를 설명하는 숨은 원인.
  "어느 클러스터", "어떤 토픽", "이미지의 의미적 속성" 등.

주변화:
  P(x) = Σ_z P(x,z) = Σ_z P(x|z)·P(z)
  z를 합산(적분)하여 제거하고 x만의 확률을 구한다.

왜 어려운가:
  이 적분이 대부분 풀리지 않는다 (고차원).
  → EM: z의 사후분포를 추정하며 번갈아 최적화
  → 변분 추론: P(z|x)를 Q(z|x)로 근사 → VAE
  → MCMC: 사후분포에서 샘플링

핵심 연결:
  조건부 확률 → 전확률 정리 → 주변화 → 잠재 변수 모델
  → EM / VAE / 확산 모델
```
