모든 건 0,1 

두대의 컴퓨터가 통신 0과 1을 주고 받을 수 있다.

전달하면 전선을 통함. 디지털 신호를 아날로그로 보내고 아날로그를 디지털로 주고받아서 성공.

이 과정을 해주는 게 예전에 모뎀. 한 컴퓨터에서 디지털을 아날로그로 변환해서, 다른 컴퓨터 받는 컴퓨터는 아날로그를 디지털로 전환해서 디코딩한다. 이게 피지컬 레이어, 하드웨어.

## 여러 대의 컴퓨터의 통신

하나의 전선으로 여러 대의 컴퓨터가 연결되어 있다.

그럼 이게 내가 보내려는 상대가 정해지는 게 아니라 연결되어 있는 전선에 컴퓨터가 모두 전송됨. 이게 문제임.

메시지의 목적지를 확인해서 특정 사람한테만 전송하게 가능한 것 → **스위치**

스위치도 일종의 컴퓨터. 스위치라는 하나의 네트워크. 스위치끼리도 연결되어 있음.

이걸 연결해주는 게 **라우터**. 라우터를 라우터끼리 연결할 수도 있음. 인터넷 자체를 거대한 라우터들끼리의 집합으로 설명하는 사람도 있음.

이런 건 실제 해수면 아래에 해저 케이블 / 스타링크. 여기까지가 물리 계층.

## 데이터 링크 계층

데이터는 0101010101010로 옴. 이거를 잘 끊어 읽어야 함.

이를 위해서 송신자는 데이터 앞뒤에 특정한 비트열을 붙인다 → **Data Link Layer**

같은 네트워크에 있는 여러 대의 컴퓨터들이 데이터를 주고받는 데 필요한 모듈.

Framing 같은 작업이 원본 데이터를 감쌈. 이거 컴퓨터의 **랜카드(NIC)**임. 여기까지 하드웨어적으로 구성됨.

## 네트워크 계층 (3계층)

라우터 내부에서 어느 라우터로 보내야 되는지 알고 있을까?

패킷으로 포장해서 계속 넘겨줌. 어딘가의 목적지로 데이터를 전송하기 위해 IP 주소를 이용해서 (routing) 자신의 다음 라우터에게 데이터를 넘겨주는 것.

운영체제 커널에 소프트웨어적으로 구현.

## 전송 계층 (4계층)

컴퓨터에는 여러 개의 프로그램들이 실행되고 있습니다.

받은 데이터들을 프로세스에게 제대로 나눠줘야 됨. 데이터를 받고자 하는 프로세스는 **포트번호**로 식별. 데이터 전송자가 포트번호까지 알아야 한다.

얘도 커널에 소프트웨어적으로 구현.

## 응용 계층 (Application Layer)

세션 레이어 프레젠테이션 레이어는 건너뜀.

**TCP/IP 소켓 프로그래밍** — 운영체제의 Transport Layer에서 제공하는 API를 활용해서 통신 가능한 프로그램을 만드는 것.

대표적 프로토콜: HTTP, 디코딩/인코딩 등.
