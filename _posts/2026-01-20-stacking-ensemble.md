# Level 1: 스태킹의 기본 원리

## 왜 서로 다른 모델을 조합하는가

모든 모델에는 구조적 한계가 있다. 각 모델은 데이터의 서로 다른 측면을 잘 포착한다.

```
랜덤 포레스트 (RF):
  강점 -- 비선형 패턴, 특성 간 복잡한 상호작용
  약점 -- 외삽(extrapolation) 불가, 선형 관계는 비효율적으로 학습

로지스틱 회귀 (LR):
  강점 -- 선형 관계를 정확하게 포착, 확률 해석 가능
  약점 -- 비선형 패턴을 잡지 못함 (특성 엔지니어링 없이는)

KNN:
  강점 -- 국소 패턴(local pattern), 복잡한 결정 경계
  약점 -- 고차원에서 성능 저하, 전역 패턴을 놓침

XGBoost:
  강점 -- 순차적 보정으로 높은 정확도
  약점 -- 범주형 특성 처리, 선형 관계에서 불필요한 복잡도
```

핵심 아이디어: **이 모델들은 "다른 실수"를 한다.**

```
예제: 이탈 예측 (실제 정답: 이탈)

고객 A:
  RF:      이탈 (맞음)    -- 비선형 소비 패턴을 포착
  LR:      잔류 (틀림)    -- 선형적으로는 잔류 고객과 비슷
  KNN:     이탈 (맞음)    -- 주변 이웃이 이탈 고객
  XGBoost: 이탈 (맞음)    -- 잔차 보정으로 잡아냄

고객 B:
  RF:      잔류 (틀림)    -- 트리 분할로는 잡기 어려운 패턴
  LR:      이탈 (맞음)    -- 선형 추세가 명확
  KNN:     잔류 (틀림)    -- 이웃 데이터가 혼재
  XGBoost: 이탈 (맞음)    -- 부스팅으로 보정

단일 모델 최고 정확도: 75%
다수결 조합:           82%
메타 모델 조합:        85%
```

RF가 틀릴 때 LR이 맞추고, LR이 틀릴 때 XGBoost가 맞추는 식이다. 이 상보적(complementary) 관계를 체계적으로 활용하는 것이 스태킹이다.

## Level 0 / Level 1 아키텍처

스태킹은 2층 구조다. Level 0의 개별 모델(base learner)이 예측을 만들고, Level 1의 메타 모델(meta learner)이 그 예측들을 조합한다.

```
입력 데이터 X
     |
     +---> [모델 A: RF]           ---> 예측 p_A
     |
     +---> [모델 B: XGBoost]      ---> 예측 p_B
     |
     +---> [모델 C: 로지스틱 회귀] ---> 예측 p_C
     |
     +---> [모델 D: KNN]          ---> 예측 p_D
     |
     v
Level 0 출력: [p_A, p_B, p_C, p_D]
     |
     v
[메타 모델: 로지스틱 회귀 또는 릿지]
     |
     v
최종 예측 y_hat
```

메타 모델은 "어떤 상황에서 어떤 모델의 예측을 더 신뢰할지"를 학습한다. 단순 평균보다 똑똑하다.

## 숫자로 따라가는 스태킹 예제

분류 문제: 4개의 샘플에 대해 4개 모델이 확률 예측을 한다.

```
         모델A(RF)  모델B(XGB)  모델C(LR)  모델D(KNN)  실제값
샘플 1:    0.82       0.91        0.65        0.88        1
샘플 2:    0.35       0.28        0.72        0.31        0
샘플 3:    0.68       0.74        0.55        0.79        1
샘플 4:    0.15       0.22        0.48        0.12        0

단순 평균:
  샘플 1: (0.82+0.91+0.65+0.88)/4 = 0.815 --> 1 (맞음)
  샘플 2: (0.35+0.28+0.72+0.31)/4 = 0.415 --> 0 (맞음)
  샘플 3: (0.68+0.74+0.55+0.79)/4 = 0.690 --> 1 (맞음)
  샘플 4: (0.15+0.22+0.48+0.12)/4 = 0.243 --> 0 (맞음)

단순 평균도 나쁘지 않다. 그런데 모델C(LR)를 보면:
  샘플 2에서 0.72로 혼자 틀린 방향을 가리킨다.
  샘플 4에서 0.48로 혼자 애매하다.
  --> 모델C의 가중치를 낮추면 더 좋지 않을까?

메타 모델이 학습한 가중치 (예시):
  모델A: 0.25, 모델B: 0.35, 모델C: 0.10, 모델D: 0.30

가중 평균:
  샘플 2: 0.25*0.35 + 0.35*0.28 + 0.10*0.72 + 0.30*0.31
        = 0.088 + 0.098 + 0.072 + 0.093 = 0.351
  --> 0.415보다 0에 더 가까워짐. 더 확신 있는 예측!
```

메타 모델은 단순 가중 평균을 넘어서, 비선형적인 조합도 학습할 수 있다. 예를 들어 "모델A가 0.5 이상이고 모델B가 0.8 이상이면 거의 확실히 양성"과 같은 규칙을 암묵적으로 학습한다.

## 왜 학습 데이터의 예측을 그대로 쓰면 안 되는가 (데이터 누수)

```
잘못된 방법:

1. 전체 학습 데이터로 모델A, B, C, D를 학습
2. 같은 학습 데이터에 대해 예측 --> p_A, p_B, p_C, p_D
3. 이 예측들로 메타 모델을 학습

문제: 모델A가 학습 데이터를 외웠다면?
  --> p_A가 거의 완벽한 예측 (학습 정확도 99%)
  --> 메타 모델은 "모델A만 믿으면 된다"고 학습
  --> 테스트 데이터에서 모델A가 75%밖에 못 맞추면?
  --> 메타 모델도 함께 무너진다

이것이 데이터 누수(data leakage)다.
Level 0 모델이 "이미 본 데이터"에 대한 예측을 메타 모델에 넘기면,
메타 모델은 과적합된 신호를 진짜 신호로 착각한다.
```

해결책: **모델이 "본 적 없는" 데이터에 대한 예측을 만들어야 한다.** 이것이 K-fold 스태킹이다.

# Level 2: 스태킹 구현

## K-fold 스태킹: 상세 절차

5-fold 스태킹의 전체 과정을 따라가 본다. 학습 데이터 1000개, 테스트 데이터 200개, 베이스 모델 3개(RF, XGBoost, LR)라 하자.

```
학습 데이터: 1000개 --> 5개 폴드로 나눔 (각 200개)
테스트 데이터: 200개

=== 모델 A (RF)의 OOF 예측 생성 ===

Fold 1:
  학습: 폴드 2,3,4,5 (800개) --> RF 학습
  예측: 폴드 1 (200개)       --> OOF 예측 [p1...p200]
  테스트 예측: 200개          --> test_pred_1

Fold 2:
  학습: 폴드 1,3,4,5 (800개) --> RF 학습
  예측: 폴드 2 (200개)       --> OOF 예측 [p201...p400]
  테스트 예측: 200개          --> test_pred_2

Fold 3:
  학습: 폴드 1,2,4,5 (800개) --> RF 학습
  예측: 폴드 3 (200개)       --> OOF 예측 [p401...p600]
  테스트 예측: 200개          --> test_pred_3

Fold 4:
  학습: 폴드 1,2,3,5 (800개) --> RF 학습
  예측: 폴드 4 (200개)       --> OOF 예측 [p601...p800]
  테스트 예측: 200개          --> test_pred_4

Fold 5:
  학습: 폴드 1,2,3,4 (800개) --> RF 학습
  예측: 폴드 5 (200개)       --> OOF 예측 [p801...p1000]
  테스트 예측: 200개          --> test_pred_5

결과:
  OOF 예측: [p1...p1000]  -- 1000개 (학습 데이터 전체에 대한 "본 적 없는" 예측)
  테스트 예측: (test_pred_1 + ... + test_pred_5) / 5  -- 5개 모델의 평균
```

```
=== 같은 과정을 모델 B (XGBoost), 모델 C (LR)에도 반복 ===

최종적으로 메타 모델의 입력이 되는 행렬:

            모델A(RF)  모델B(XGB)  모델C(LR)
            OOF예측     OOF예측     OOF예측
샘플 1:      0.72       0.81        0.65
샘플 2:      0.35       0.28        0.42
...
샘플 1000:   0.88       0.91        0.79

이 (1000 x 3) 행렬이 메타 모델의 학습 데이터.
각 행의 레이블은 원래 학습 데이터의 y값.

테스트 데이터도 같은 형태:
            모델A(RF)  모델B(XGB)  모델C(LR)
            평균예측    평균예측     평균예측
테스트 1:    0.68       0.75        0.60
...
테스트 200:  0.22       0.18        0.30

이 (200 x 3) 행렬이 메타 모델의 테스트 데이터.
```

```
=== 메타 모델 학습 ===

메타 모델 (로지스틱 회귀):
  입력: (1000 x 3) OOF 예측 행렬
  레이블: 원래 y값 (1000개)
  --> 학습

메타 모델로 테스트 데이터 예측:
  입력: (200 x 3) 테스트 예측 행렬
  --> 최종 예측 (200개)
```

데이터 흐름을 그림으로 정리하면:

```
[학습 데이터 1000개]
        |
   5-fold 분할
   /    |    \
 RF   XGB   LR        <-- Level 0: 각 모델이 5번 학습/예측
   \    |    /
    v   v   v
[OOF 예측 행렬]         <-- (1000 x 3)
   +  [레이블 y]
        |
  [메타 모델 학습]       <-- Level 1
        |
        v
[테스트 예측 행렬]       <-- (200 x 3), 각 모델의 5-fold 평균
        |
  [메타 모델 예측]
        |
        v
   [최종 결과]
```

### 코드로 보는 K-fold 스태킹

```python
import numpy as np
from sklearn.model_selection import KFold
from sklearn.ensemble import RandomForestClassifier
from sklearn.linear_model import LogisticRegression
from xgboost import XGBClassifier

# 베이스 모델 정의
base_models = [
    RandomForestClassifier(n_estimators=200, random_state=42),
    XGBClassifier(n_estimators=200, learning_rate=0.1, random_state=42),
    LogisticRegression(max_iter=1000)
]

n_folds = 5
kf = KFold(n_splits=n_folds, shuffle=True, random_state=42)

# OOF 예측 저장할 배열
oof_predictions = np.zeros((len(X_train), len(base_models)))
test_predictions = np.zeros((len(X_test), len(base_models)))

for i, model in enumerate(base_models):
    test_preds_folds = np.zeros((len(X_test), n_folds))

    for fold_idx, (train_idx, val_idx) in enumerate(kf.split(X_train)):
        X_tr, X_val = X_train[train_idx], X_train[val_idx]
        y_tr, y_val = y_train[train_idx], y_train[val_idx]

        model.fit(X_tr, y_tr)

        # OOF 예측 (본 적 없는 데이터에 대한 예측)
        oof_predictions[val_idx, i] = model.predict_proba(X_val)[:, 1]

        # 테스트 예측 (나중에 폴드별 평균)
        test_preds_folds[:, fold_idx] = model.predict_proba(X_test)[:, 1]

    # 테스트 예측은 5개 폴드의 평균
    test_predictions[:, i] = test_preds_folds.mean(axis=1)

# 메타 모델 학습
meta_model = LogisticRegression()
meta_model.fit(oof_predictions, y_train)

# 최종 예측
final_predictions = meta_model.predict(test_predictions)
```

## 블렌딩 vs 스태킹

블렌딩(Blending)은 스태킹의 단순화 버전이다. K-fold 대신 홀드아웃 셋을 사용한다.

```
스태킹 (K-fold):
  학습 데이터 전체를 K번 나눠서 OOF 예측 생성
  --> 모든 학습 데이터에 대한 메타 특성을 만듦
  --> 데이터를 최대한 활용

블렌딩 (Holdout):
  학습 데이터를 한 번만 나눔 (예: 80% / 20%)
  80%로 Level 0 모델 학습
  20%에 대한 예측으로 메타 모델 학습
  --> 구현이 단순하고 과적합 위험이 낮음
  --> 하지만 메타 모델이 20% 데이터만으로 학습 (데이터 낭비)
```

```
블렌딩 절차:

[학습 데이터]
   |
   +---> [학습용 80%] --> RF, XGB, LR 학습
   |
   +---> [블렌딩용 20%] --> 학습된 모델로 예측
                              |
                         [예측 행렬]
                              |
                       [메타 모델 학습]
                              |
                       [테스트 예측]
```

**블렌딩이 더 나은 경우:**

```
1. 데이터가 매우 클 때 (100만건 이상)
   --> K-fold 스태킹은 모델을 K*M번 학습해야 함 (K: 폴드 수, M: 모델 수)
   --> 블렌딩은 M번만 학습
   --> 시간 절약이 극적

2. 과적합이 걱정될 때
   --> K-fold 스태킹은 정보 누수가 미묘하게 발생할 수 있음
   --> 블렌딩은 완전히 분리된 데이터를 사용하므로 깨끗함

3. 빠른 실험이 필요할 때
   --> 구현이 간단하고 디버깅이 쉬움
```

**스태킹이 더 나은 경우:**

```
1. 데이터가 적을 때 (수천 ~ 수만 건)
   --> 20%를 떼어내면 메타 모델 학습 데이터가 너무 적음
   --> K-fold로 전체 데이터를 활용해야 함

2. 최대 성능이 필요할 때 (대회)
   --> OOF 예측이 더 안정적
   --> 더 많은 메타 학습 데이터
```

## 메타 모델 선택: 왜 단순한 모델이 좋은가

메타 모델은 **단순할수록 좋다.** 그 이유:

```
메타 모델의 입력은 무엇인가?
  --> Level 0 모델들의 예측값 (3~10개 정도)
  --> 이미 고도로 가공된 특성이다

이 3~10개의 입력으로 복잡한 모델을 학습하면?
  --> 특성 수가 너무 적어서 과적합
  --> 랜덤 포레스트를 메타 모델로 쓰면, 3개 특성에 트리 100개
  --> 과적합이 거의 확실

좋은 메타 모델:
  로지스틱 회귀  -- 선형 조합, 과적합 위험 낮음
  릿지 회귀      -- L2 정규화로 가중치 제한
  선형 SVM       -- 마진 최대화

피해야 할 메타 모델:
  깊은 신경망    -- 3개 입력에 1000개 파라미터? 과적합
  랜덤 포레스트  -- 불필요하게 복잡
  XGBoost       -- Level 0에서 이미 쓰고 있다면 중복

예외:
  Level 0 모델이 10개 이상이고, 원본 특성도 메타 입력에 포함한다면
  비선형 메타 모델이 도움이 될 수 있다.
  하지만 이 경우에도 정규화가 강하게 필요하다.
```

## 다층 스태킹: Level 0 --> Level 1 --> Level 2

스태킹을 여러 층으로 쌓을 수 있다.

```
Level 0:  RF, XGBoost, LR, KNN, SVM
              |
              v
Level 1:  LightGBM (Level 0의 예측 + 원본 특성 일부를 입력으로)
              |
              v
Level 2:  로지스틱 회귀 (Level 1의 예측을 입력으로)
              |
              v
          최종 예측

각 레벨에서 K-fold OOF 예측을 사용한다.
```

**수확 체감의 법칙:**

```
레벨 수    성능 향상    복잡도    학습 시간
───────────────────────────────────────────
Level 1      +3%        낮음     x1
Level 2      +1%        중간     x3
Level 3      +0.2%      높음     x9

대부분의 경우 2-level 스태킹이 최적이다.
3-level 이상은 Kaggle 상위 솔루션에서나 볼 수 있고,
실무에서는 복잡도 대비 이득이 거의 없다.
```

# Level 3: 앙상블 전략 가이드

## 배깅 vs 부스팅 vs 스태킹: Bias-Variance 관점

```
                배깅               부스팅              스태킹
────────────────────────────────────────────────────────────────────
핵심 원리      독립적 모델의 평균   순차적 잔차 보정    이질적 모델 조합
해결하는 문제   높은 Variance       높은 Bias           Bias + Variance
기본 모델      깊은 트리 (복잡)     얕은 트리 (단순)    다양한 유형
모델 관계      병렬 (독립)          직렬 (순차)         2단계 (계층)
대표 알고리즘   랜덤 포레스트       XGBoost, LightGBM   --
과적합 위험     낮음               중간                높음
학습 속도      병렬화 가능         순차적 (느림)       모델 수에 비례
해석 가능성     특성 중요도 가능    SHAP으로 가능       거의 불가능
튜닝 난이도     쉬움               중간 (많은 HP)      어려움

Bias-Variance 작동 원리:

배깅:
  깊은 트리 = 낮은 Bias + 높은 Variance
  여러 개의 평균 --> Variance가 1/n으로 감소
  Bias는 그대로 낮게 유지
  --> 결과: 낮은 Bias + 낮은 Variance

부스팅:
  얕은 트리 = 높은 Bias + 낮은 Variance
  순차적 보정 --> Bias가 점차 감소
  학습률(eta)로 Variance 증가를 제어
  --> 결과: 낮은 Bias + 낮은 Variance

스태킹:
  다양한 모델 = 각기 다른 Bias-Variance 프로파일
  메타 모델이 최적 조합을 학습
  --> 결과: 각 모델의 장점을 취합
```

## 다양성이 핵심이다: 모델 간 상관관계

앙상블이 효과를 발휘하려면 모델들이 **서로 다른 실수**를 해야 한다.

```
극단적 예시 1: 다양성이 없는 앙상블

  모델 A: RF (seed=42)
  모델 B: RF (seed=42, 같은 하이퍼파라미터)
  모델 C: RF (seed=42, 같은 하이퍼파라미터)

  세 모델이 완전히 같은 예측 --> 앙상블해도 단일 모델과 동일
  상관계수 = 1.0
  개선 효과 = 0

극단적 예시 2: 다양성이 높은 앙상블

  모델 A: RF (비선형 패턴에 강함)
  모델 B: 로지스틱 회귀 (선형 패턴에 강함)
  모델 C: KNN (국소 패턴에 강함)

  각 모델이 다른 곳에서 틀림 --> 조합하면 상호 보완
  상관계수 = 0.3~0.5
  개선 효과 = 크다
```

**다양성을 측정하는 방법:**

```
1. Q-통계량 (Yule's Q):
   두 분류기 i, j의 예측 결과표:

              분류기 j 맞음    분류기 j 틀림
   분류기 i 맞음    N11           N10
   분류기 i 틀림    N01           N00

   Q_ij = (N11*N00 - N01*N10) / (N11*N00 + N01*N10)

   Q = 1:   두 분류기가 완전히 같은 패턴으로 맞고 틀림
   Q = 0:   통계적으로 독립
   Q = -1:  완전히 반대 패턴

   좋은 앙상블: Q가 낮을수록 좋다 (0에 가까울수록)

2. 불일치율 (Disagreement Measure):
   D_ij = (N01 + N10) / (N11 + N10 + N01 + N00)

   두 분류기가 서로 다른 예측을 한 비율.
   D가 높을수록 다양성이 높다.

3. 실전에서의 간편한 방법:
   각 모델의 예측 확률 간 피어슨 상관계수를 구한다.

   모델 조합        상관계수     앙상블 효과
   ─────────────────────────────────────────
   RF + RF2          0.95        거의 없음
   RF + XGBoost      0.80        약간 있음
   RF + LR           0.55        꽤 있음
   XGBoost + KNN     0.45        좋음
   RF + LR + KNN     평균 0.50   매우 좋음

   경험 법칙: 상관계수가 0.7 이하인 모델들을 조합하면 효과적.
```

**다양성을 높이는 전략:**

```
1. 알고리즘 다양성:
   트리 기반 + 선형 모델 + 거리 기반 + 신경망

2. 데이터 다양성:
   서로 다른 특성 부분집합으로 학습
   예: 모델A는 수치 특성, 모델B는 범주 특성, 모델C는 전체

3. 하이퍼파라미터 다양성:
   같은 알고리즘이라도 설정을 다르게
   예: RF(100 trees, depth=10) + RF(500 trees, depth=5)

4. 전처리 다양성:
   표준화된 데이터로 LR, 원본 데이터로 RF, PCA 데이터로 KNN
```

## 투표 전략 (Voting)

스태킹보다 단순한 앙상블 방법들이다. 메타 모델 없이 예측을 조합한다.

### 경성 투표 (Hard Voting): 다수결

```
3개 모델의 클래스 예측:

           모델A    모델B    모델C    다수결
샘플 1:      1        1        0        1    (2:1)
샘플 2:      0        0        0        0    (3:0)
샘플 3:      1        0        1        1    (2:1)
샘플 4:      0        1        0        0    (2:1)

단순하고 직관적이다.
단점: 예측의 확신도를 무시한다.
```

### 연성 투표 (Soft Voting): 확률 평균

```
3개 모델의 확률 예측:

           모델A    모델B    모델C    평균     판정
샘플 1:     0.90     0.85     0.40    0.717     1
샘플 2:     0.10     0.15     0.60    0.283     0
샘플 3:     0.70     0.45     0.80    0.650     1
샘플 4:     0.30     0.51     0.20    0.337     0

샘플 2를 보자:
  경성 투표: 모델A(0), 모델B(0), 모델C(1) --> 다수결 0
  연성 투표: 0.283 --> 0

  같은 결과지만, 연성 투표는 "얼마나 확신하는가"까지 반영.

샘플 3을 보자:
  경성 투표: 모델B가 0.45로 클래스 0 예측 --> 투표 2:1 = 1
  연성 투표: 0.45는 0에 가깝지만 크게 틀리진 않음, 평균 0.65 --> 1

  만약 모델B가 0.49였다면?
  경성 투표에서는 클래스 0이지만, 실제로는 거의 반반.
  연성 투표가 이런 경계 사례를 더 잘 처리한다.

결론: 모델이 확률을 출력할 수 있다면 연성 투표가 거의 항상 더 좋다.
```

### 가중 투표 (Weighted Voting): 검증 성능 기반

```
검증 데이터에서의 각 모델 성능:
  모델A (RF):      AUC 0.85
  모델B (XGBoost):  AUC 0.90
  모델C (LR):       AUC 0.78

가중치 계산 (성능 비례):
  총합 = 0.85 + 0.90 + 0.78 = 2.53
  w_A = 0.85/2.53 = 0.336
  w_B = 0.90/2.53 = 0.356
  w_C = 0.78/2.53 = 0.308

가중 연성 투표:
  y_hat = 0.336 * p_A + 0.356 * p_B + 0.308 * p_C

XGBoost의 예측에 가장 높은 가중치.
성능이 낮은 LR의 가중치는 상대적으로 낮다.

주의: 가중치를 검증 데이터로 정했으면,
      최종 평가는 반드시 별도의 테스트 데이터로 해야 한다.
      그렇지 않으면 가중치 자체가 과적합된다.
```

```python
# scikit-learn 구현
from sklearn.ensemble import VotingClassifier

# 연성 투표
soft_voter = VotingClassifier(
    estimators=[
        ('rf', RandomForestClassifier(n_estimators=200)),
        ('xgb', XGBClassifier(n_estimators=200)),
        ('lr', LogisticRegression())
    ],
    voting='soft',
    weights=[0.336, 0.356, 0.308]  # 가중치 (선택)
)
soft_voter.fit(X_train, y_train)
predictions = soft_voter.predict(X_test)
```

## 앙상블을 쓰지 말아야 할 때

앙상블이 항상 정답은 아니다.

```
1. 지연 시간(Latency) 제약이 있을 때

   단일 LightGBM 추론: 0.1ms
   5개 모델 스태킹:    0.5ms + 메타 모델 0.05ms = 0.55ms

   5배 느려진다. 실시간 서빙(광고 입찰, 추천 등)에서는 치명적.
   모델 5개를 메모리에 올려야 하므로 메모리도 5배.

   대안: 단일 모델을 잘 튜닝하거나, 앙상블 결과를 증류(distillation)해서
        단일 모델에 옮긴다.

2. 해석 가능성이 필요할 때

   금융 규제: "왜 대출을 거절했는가?"를 설명해야 함
   의료: "왜 이 진단을 내렸는가?"
   법률: 차별 방지를 위한 모델 감사(audit)

   스태킹의 메타 모델이 "RF가 0.7이고 XGB가 0.8이라서 승인"은
   비전문가에게 설명할 수 없다.

   대안: 단일 트리 + 가지치기, 또는 단일 모델 + SHAP

3. 한계 개선이 복잡도를 정당화하지 못할 때

   AUC 0.920 (LightGBM 단독)
   AUC 0.925 (5모델 스태킹)

   0.005 향상을 위해:
   - 5개 모델 학습/유지보수
   - 모델 버전 관리 5배 복잡
   - 배포 파이프라인 5배 복잡
   - 장애 지점 5배 증가
   - 새 팀원이 시스템을 이해하는 시간 3배

   비즈니스 임팩트를 따져보면 대부분 단일 모델이 낫다.

4. 데이터가 계속 변할 때 (concept drift)

   모델을 자주 재학습해야 하는 환경에서
   5개 모델을 모두 재학습하고 메타 모델도 다시 학습하는 것은
   운영 부담이 크다.
```

## Kaggle 상위 솔루션 분석: 공통 패턴

Kaggle 대회에서 반복적으로 나타나는 앙상블 패턴을 정리한다.

```
=== 패턴 1: 다양한 시드 앙상블 ===

같은 모델을 다른 시드(seed)로 여러 번 학습하고 평균.

LightGBM(seed=0) + LightGBM(seed=1) + ... + LightGBM(seed=9)
--> 10개의 평균

효과: Variance 감소. 가장 쉬운 앙상블.
상위 솔루션의 거의 모든 제출에 사용된다.
성능 향상: +0.1~0.5% (작지만 안정적)

=== 패턴 2: 폴드별 모델 활용 ===

5-fold CV로 학습하면 5개 모델이 생긴다.
이 5개의 예측을 평균하면 그 자체가 앙상블.

실질적으로 모든 상위 솔루션이 이것을 사용.

=== 패턴 3: 2~3종 부스팅 + 1~2종 신경망 ===

전형적 상위 솔루션 구성:

Level 0:
  LightGBM (2~3개, 다른 하이퍼파라미터)
  XGBoost (1~2개)
  CatBoost (1~2개, 범주형 특성이 있으면)
  TabNet 또는 간단한 NN (1개, 다양성 확보용)

Level 1:
  릿지 회귀 또는 로지스틱 회귀

총 모델 수: 5~10개

=== 패턴 4: 후처리(Post-processing) ===

앙상블 이후에도 추가 트릭:
  - 예측 분포 조정 (테스트 데이터의 분포에 맞춤)
  - 임계값 최적화 (0.5가 아닌 최적 cutoff 탐색)
  - 순위 평균 (rank averaging): 확률 대신 순위를 평균
    --> 모델 간 스케일 차이 문제를 해결

=== 패턴 5: 순위 평균 (Rank Averaging) ===

모델마다 확률의 스케일이 다를 수 있다:
  모델A: 확률이 0.1~0.9 범위
  모델B: 확률이 0.3~0.7 범위 (보수적)

단순 평균하면 모델A에 끌려간다.

해결: 확률을 순위로 변환 후 평균

  모델A 예측:  [0.9, 0.1, 0.5, 0.7]
  모델A 순위:  [4,   1,   2,   3]   (순위를 0~1로 정규화: [1.0, 0.0, 0.33, 0.67])

  모델B 예측:  [0.7, 0.3, 0.6, 0.5]
  모델B 순위:  [4,   1,   3,   2]   (정규화: [1.0, 0.0, 0.67, 0.33])

  순위 평균:   [1.0, 0.0, 0.50, 0.50]

  --> 스케일에 무관하게 "상대적 순서"를 보존한 앙상블
```

```
=== 상위 솔루션의 공통 교훈 ===

1. 특성 엔지니어링이 앙상블보다 중요하다
   좋은 특성 1개 >> 모델 5개 추가

2. CV 전략이 리더보드 점수를 결정한다
   적절한 CV가 없으면 앙상블이 과적합될 수 있다

3. 단순한 앙상블부터 시작한다
   연성 투표 --> 가중 투표 --> 스태킹 순서로

4. 최종 제출 직전에 시드 앙상블을 추가한다
   가장 안전한 성능 향상 방법
```

## 이 글에서 가져가야 할 것

```
Level 1:
  서로 다른 모델은 "다른 실수"를 한다
  스태킹은 Level 0(베이스 모델)의 예측을 Level 1(메타 모델)이 조합
  학습 데이터의 예측을 그대로 쓰면 데이터 누수 발생

Level 2:
  K-fold OOF 예측으로 데이터 누수를 방지한다
  블렌딩은 단순하고 안전한 대안 (홀드아웃 사용)
  메타 모델은 단순할수록 좋다 (로지스틱 회귀, 릿지)

Level 3:
  배깅은 Variance를, 부스팅은 Bias를, 스태킹은 둘 다 해결
  앙상블의 핵심은 모델 다양성 -- 상관계수가 낮을수록 효과적
  연성 투표가 경성 투표보다 거의 항상 좋다
  실무에서는 복잡도/지연/해석 가능성을 고려하여 앙상블 수준을 결정
```
