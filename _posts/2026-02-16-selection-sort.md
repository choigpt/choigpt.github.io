## 알고리즘

선택 정렬은 배열을 **정렬된 부분**과 **정렬되지 않은 부분**으로 나누고, 매 단계마다 정렬되지 않은 부분에서 최솟값을 찾아 정렬된 부분의 끝에 놓는다.

1. 전체 배열에서 최솟값을 찾는다.
2. 최솟값을 배열의 첫 번째 원소와 교환한다.
3. 첫 번째 원소를 제외한 나머지 배열에서 최솟값을 찾는다.
4. 최솟값을 두 번째 위치와 교환한다.
5. 배열 끝까지 이 과정을 반복한다.

---

## 구현

```java
void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        if (minIdx != i) {
            int temp = arr[i];
            arr[i] = arr[minIdx];
            arr[minIdx] = temp;
        }
    }
}
```

---

## 시간 복잡도

최선, 평균, 최악 모두 O(n^2)이다. 이미 정렬되어 있어도 최솟값을 찾기 위해 전부 탐색하며, 비교 횟수는 n(n-1)/2이다.

버블 정렬과 달리 **조기 종료가 불가능**하다. 정렬 여부와 관계없이 항상 같은 횟수만큼 비교한다.

---

## 공간 복잡도

O(1) -- 추가 배열 없이 주어진 배열 안에서 교환만 수행하는 **제자리 정렬(in-place)**이다.

---

## 불안정 정렬 (Unstable Sort)

선택 정렬은 **불안정 정렬**이다. 교환 과정에서 같은 값을 가진 원소의 상대적 순서가 바뀔 수 있다.

예시 1 (이 케이스는 우연히 순서가 유지됨): `[3a, 2, 3b]`
- 1단계: 최솟값 `2`를 `3a`와 교환 -> `[2, 3a, 3b]`
- 결과적으로 `3a`와 `3b`의 상대 순서는 그대로다.

예시 2 (불안정성이 드러나는 케이스): `[2a, 2b, 1]`
- 1단계: 최솟값 `1`을 `2a`와 교환 -> `[1, 2b, 2a]`
- `2a`와 `2b`의 상대 순서가 바뀌었다 — 그래서 선택 정렬은 불안정 정렬로 분류된다.

---

## 버블 정렬과 비교

버블 정렬은 시간 복잡도가 O(n^2)이지만 최선의 경우 O(n)이고, 교환 횟수는 최대 O(n^2)이며, 안정 정렬이고, 조기 종료가 가능하다. 반면 선택 정렬은 항상 O(n^2)이고, 교환 횟수는 최대 n-1이며, 불안정 정렬이고, 조기 종료가 불가능하다.

---

## 언제 쓸 수 있는가

선택 정렬의 장점은 **교환 횟수가 최대 n-1번**이라는 것이다. 비교는 많이 하지만 실제 데이터 이동은 적다. 따라서 쓰기(write) 비용이 비교(compare) 비용보다 훨씬 큰 환경(예: 플래시 메모리)에서는 버블 정렬보다 유리할 수 있다.

그 외 일반적인 상황에서는 O(n log n) 정렬 알고리즘이 사실상 항상 더 빠르다.
