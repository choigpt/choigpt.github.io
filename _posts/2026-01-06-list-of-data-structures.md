## 배열 (Array)

배열은 **연속된 메모리 공간**에 동일한 타입의 원소를 저장하는 자료구조다.

- **인덱스 접근**: O(1) -- 시작 주소 + (인덱스 x 원소 크기)로 바로 계산
- **삽입/삭제**: O(n) -- 중간에 넣거나 빼면 뒤의 원소를 전부 밀거나 당겨야 한다
- **크기 고정**: 선언 시 크기가 정해지며, 이후 변경 불가

### 동적 배열 (Dynamic Array)

Java의 `ArrayList`, Python의 `list`가 대표적이다. 내부적으로 고정 크기 배열을 사용하되, 용량이 가득 차면 더 큰 배열을 새로 할당하고 기존 원소를 복사한다. 이 때문에 `add` 연산은 **분할 상환(amortized) O(1)**이다.

## 연결 리스트 (Linked List)

각 원소(노드)가 **데이터와 다음 노드의 포인터**를 갖는 구조다. 메모리상 연속일 필요가 없다.

| 종류 | 특징 |
|------|------|
| 단일 연결 리스트 | 각 노드가 다음 노드만 가리킨다 |
| 이중 연결 리스트 | 이전 노드와 다음 노드 모두 가리킨다 |
| 원형 연결 리스트 | 마지막 노드가 첫 번째 노드를 가리킨다 |

- **삽입/삭제**: 위치를 알고 있으면 O(1) -- 포인터만 바꾸면 된다
- **탐색**: O(n) -- 인덱스 접근이 불가능하므로 처음부터 순회해야 한다
- **메모리 오버헤드**: 포인터 저장 공간이 추가로 필요하다

## 스택 (Stack)

**LIFO**(Last In, First Out) 구조다. 마지막에 넣은 원소가 가장 먼저 나온다.

- **push**: 스택 최상단에 원소 추가 -- O(1)
- **pop**: 스택 최상단 원소 제거 및 반환 -- O(1)
- **peek/top**: 최상단 원소 확인 -- O(1)

### 사용처

- 함수 호출 스택 (call stack)
- 실행 취소(undo) 기능
- 괄호 매칭, 후위 표기식 계산
- DFS 구현

---

## 큐 (Queue)

**FIFO**(First In, First Out) 구조다. 먼저 넣은 원소가 먼저 나온다.

- **enqueue**: 큐의 뒤쪽에 원소 추가 -- O(1)
- **dequeue**: 큐의 앞쪽 원소 제거 및 반환 -- O(1)

### 변형

- **원형 큐**: 배열 기반 큐에서 공간 낭비를 해결한다
- **덱 (Deque)**: 양쪽 끝에서 삽입/삭제가 가능하다
- **우선순위 큐**: 우선순위가 높은 원소가 먼저 나온다 (힙으로 구현)

### 사용처

- BFS 탐색
- 작업 스케줄링 (프린터 큐, 프로세스 스케줄링)
- 메시지 큐

## 해시 테이블 (Hash Table)

**키-값(key-value) 쌍**을 저장하는 자료구조다. 해시 함수를 통해 키를 배열 인덱스로 변환한다.

- **탐색/삽입/삭제**: 평균 O(1), 최악 O(n)
- 최악의 경우는 모든 키가 같은 인덱스에 몰릴 때 발생한다

### 충돌 해결

해시 함수가 서로 다른 키에 대해 같은 인덱스를 반환하면 충돌이 발생한다.

| 방법 | 설명 |
|------|------|
| **체이닝 (Chaining)** | 같은 인덱스에 연결 리스트로 원소를 연결한다 |
| **오픈 어드레싱 (Open Addressing)** | 충돌 시 빈 슬롯을 찾아 저장한다 (선형 탐사, 이차 탐사, 이중 해싱) |

Java의 `HashMap`, Python의 `dict`가 대표적인 구현이다.

## 트리 (Tree)

**계층적 구조**를 표현하는 비선형 자료구조다. 하나의 루트 노드에서 시작하여 자식 노드로 분기한다.

### 이진 트리 (Binary Tree)

각 노드가 최대 2개의 자식을 갖는 트리다.

### 이진 탐색 트리 (BST)

이진 트리에 **정렬 규칙**을 추가한 것이다.

- 왼쪽 서브트리의 모든 값 < 부모 노드 값 < 오른쪽 서브트리의 모든 값
- 탐색/삽입/삭제: 평균 O(log n), 최악 O(n) (편향 트리)
- 균형을 유지하면 항상 O(log n)을 보장한다 (AVL 트리, 레드-블랙 트리)

### 사용처

- 파일 시스템 디렉토리 구조
- 데이터베이스 인덱스 (B-Tree, B+Tree)
- 파싱 트리 (컴파일러)

## 힙 (Heap)

**완전 이진 트리** 기반의 자료구조로, 부모-자식 간 대소 관계를 유지한다.

| 종류 | 조건 |
|------|------|
| **최소 힙 (Min Heap)** | 부모 <= 자식 -- 루트가 최솟값 |
| **최대 힙 (Max Heap)** | 부모 >= 자식 -- 루트가 최댓값 |

- **삽입**: O(log n) -- 마지막에 넣고 위로 올린다 (heapify up)
- **삭제(루트)**: O(log n) -- 루트를 제거하고 마지막 원소를 올린 뒤 아래로 내린다 (heapify down)
- **최솟값/최댓값 조회**: O(1)

배열로 구현하며, 인덱스 `i`의 자식은 `2i+1`, `2i+2`에 위치한다. 우선순위 큐의 표준 구현체다.

## 그래프 (Graph)

**정점(vertex)**과 **간선(edge)**으로 구성된 자료구조다. 트리는 그래프의 특수한 형태(사이클 없는 연결 그래프)다.

### 분류

| 기준 | 종류 |
|------|------|
| 방향 | 방향 그래프 (Directed) / 무방향 그래프 (Undirected) |
| 가중치 | 가중 그래프 (Weighted) / 비가중 그래프 (Unweighted) |

### 표현 방식

| 방식 | 공간 | 간선 존재 확인 | 특징 |
|------|------|----------------|------|
| **인접 행렬** | O(V^2) | O(1) | 간선이 많은 밀집 그래프에 유리 |
| **인접 리스트** | O(V + E) | O(degree) | 간선이 적은 희소 그래프에 유리 |

### 사용처

- 소셜 네트워크 (사용자 관계)
- 지도/경로 탐색 (네비게이션)
- 의존성 관리 (빌드 시스템, 패키지 매니저)

| 연결 리스트 | O(n) | O(n) | O(1)* | O(1)* |
| 스택 | O(n) | O(n) | O(1) | O(1) |
| 큐 | O(n) | O(n) | O(1) | O(1) |
| 해시 테이블 | - | O(1) avg | O(1) avg | O(1) avg |
| BST | - | O(log n) avg | O(log n) avg | O(log n) avg |
| 힙 | - | O(n) | O(log n) | O(log n) |

*위치를 알고 있는 경우
