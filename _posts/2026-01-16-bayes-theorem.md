이 글은 [조건부 확률](/conditional-probability/)과 [확률 변수](/random-variable/)를 전제로 한다.

- 조건부 확률 포스트에서 다룬 **곱셈 규칙**, **전확률 정리**가 베이즈 정리의 재료다.
- 확률 변수 포스트에서 다룬 **확률분포**, **기댓값**이 베이지안 추론의 언어다.

이 글도 단계적으로 깊어진다.

- **Level 1** — 베이즈 정리 유도와 직관. 공식이 말하는 것.
- **Level 2** — 반복 업데이트. 데이터가 들어올수록 사후확률이 수렴하는 과정.
- **Level 3** — 빈도주의 vs 베이지안. 실전에서의 베이지안 사고.
- **Level 4** — ML/DL로의 연결. 정규화, 생성 모델, 강화학습이 베이즈에서 나온다.

# Level 1: 베이즈 정리의 정체

## 확률을 바라보는 두 가지 관점

같은 "확률 70%"라는 말이 전혀 다른 의미를 가질 수 있다.

**빈도주의 (Frequentist)**

```
"이 동전의 앞면 확률은 0.5다"
= 이 동전을 무한히 던지면 앞면이 나오는 비율이 0.5에 수렴한다
= 반복 실험의 장기 빈도(frequency)

"내일 비 올 확률 70%"
= ??? 내일은 한 번만 온다. 반복할 수 없다.
= 빈도주의로는 해석이 어색하다.
```

**베이지안 (Bayesian)**

```
"이 동전의 앞면 확률은 0.5다"
= 내가 이 동전이 공정하다고 믿는 정도가 0.5다
= 믿음(신뢰도, degree of belief)

"내일 비 올 확률 70%"
= 현재 가진 정보(기상 데이터, 구름 상태 등)를 종합했을 때
  비가 올 것이라는 내 신뢰도가 0.7이다
= 자연스럽게 해석된다.
```

베이지안 관점에서 확률은 **"현재 가진 정보 아래에서의 불확실성"**이다. 새로운 정보(데이터)가 들어오면 확률을 **업데이트**한다. 이 업데이트 규칙이 베이즈 정리다.

## 유도: 조건부 확률에서 한 걸음

[조건부 확률](/conditional-probability/)의 곱셈 규칙에서 두 가지 형태를 세웠다:

$$P(A \cap B) = P(A \mid B) \cdot P(B)$$

$$P(A \cap B) = P(B \mid A) \cdot P(A)$$

좌변이 같으므로 우변을 등호로 연결한다:

$$P(A \mid B) \cdot P(B) = P(B \mid A) \cdot P(A)$$

양변을 $$P(B)$$로 나누면:

$$P(A \mid B) = \frac{P(B \mid A) \cdot P(A)}{P(B)}$$

이것이 **베이즈 정리**다. 조건부 확률의 방향을 뒤집는 공식이다.

## 각 항의 의미

가설(H)과 데이터(D) 표기로 바꾸면:

$$\boxed{P(H \mid D) = \frac{P(D \mid H) \cdot P(H)}{P(D)}}$$

```
사후확률 = (우도 × 사전확률) / 증거

P(H|D): 사후확률 (Posterior)
  → 데이터를 본 뒤 가설에 대한 내 믿음
  → "이것이 알고 싶은 것"

P(H): 사전확률 (Prior)
  → 데이터를 보기 전 가설에 대한 내 믿음
  → "기존 지식, 경험, 직관"

P(D|H): 우도 (Likelihood)
  → 가설이 참이라면 이 데이터가 나올 확률
  → "가설이 데이터를 얼마나 잘 설명하는가"

P(D): 증거 (Evidence)
  → 데이터가 관측될 전체 확률
  → 전확률 정리로 구한다
  → 모든 가설에 대해 같은 값이므로 "정규화 상수"라고도 한다
```

## 예제: 동전의 공정성

동전을 던졌는데 10번 중 8번이 앞면이었다. 이 동전이 공정한가?

```
가설 두 개:
  H₁: 공정한 동전 (앞면 확률 0.5)
  H₂: 편향된 동전 (앞면 확률 0.8)

사전확률: 특별한 정보 없으므로 반반
  P(H₁) = 0.5
  P(H₂) = 0.5

데이터: 10번 중 8번 앞면
  D = "10번 중 8번 앞면"

우도 (이항분포로 계산):
  P(D|H₁) = C(10,8) × 0.5⁸ × 0.5² = 45 × 0.00098 = 0.0439
  P(D|H₂) = C(10,8) × 0.8⁸ × 0.2² = 45 × 0.00671 = 0.3020

  → 공정한 동전(H₁)에서 8/10이 나올 확률: 4.4%
  → 편향된 동전(H₂)에서 8/10이 나올 확률: 30.2%
  → H₂가 이 데이터를 7배 더 잘 설명한다

증거:
  P(D) = P(D|H₁)·P(H₁) + P(D|H₂)·P(H₂)
       = 0.0439 × 0.5 + 0.3020 × 0.5
       = 0.0220 + 0.1510
       = 0.1730

사후확률:
  P(H₁|D) = 0.0439 × 0.5 / 0.1730 = 0.127  → 12.7%
  P(H₂|D) = 0.3020 × 0.5 / 0.1730 = 0.873  → 87.3%
```

```
업데이트 결과:

         사전 (데이터 전)  →  사후 (데이터 후)
공정(H₁):    50%          →     12.7%    ↓ 크게 하락
편향(H₂):    50%          →     87.3%    ↑ 크게 상승

10번 중 8번 앞면이라는 데이터가
"편향된 동전" 가설을 강하게 지지한다.
```

## 의료 검진 — 조건부 확률에서 다시 보기

[조건부 확률](/conditional-probability/)에서 10,000명 시뮬레이션으로 다룬 예제를 이번에는 베이즈 정리 공식으로 풀어보자.

```
유병률 1%, 민감도 99%, 특이도 95%
양성 판정 시 실제 환자일 확률?

H = 질병, D = 양성

P(H) = 0.01                    (사전: 유병률)
P(D|H) = 0.99                  (우도: 민감도)
P(D|¬H) = 0.05                 (위양성률 = 1 - 특이도)

P(D) = P(D|H)·P(H) + P(D|¬H)·P(¬H)    (전확률 정리)
     = 0.99 × 0.01 + 0.05 × 0.99
     = 0.0099 + 0.0495
     = 0.0594

P(H|D) = P(D|H)·P(H) / P(D)
       = 0.99 × 0.01 / 0.0594
       = 0.167

→ 16.7% (조건부 확률 포스트의 결과와 일치)
```

10,000명을 세는 방법과 베이즈 공식은 **같은 계산의 다른 표현**이다. 어떤 것이 편한지는 상황에 따라 다르다.

# Level 2: 반복 업데이트 — 데이터가 쌓이면 어떻게 되는가

## 핵심 아이디어: 사후가 다음의 사전이 된다

베이즈 정리의 진짜 힘은 **한 번의 업데이트가 아니라 반복**에 있다.

```
데이터 1차 → 사전 ──베이즈──→ 사후₁
데이터 2차 → 사후₁을 사전으로 ──베이즈──→ 사후₂
데이터 3차 → 사후₂을 사전으로 ──베이즈──→ 사후₃
...
```

이전 단계의 사후확률이 다음 단계의 사전확률이 된다. 데이터가 쌓일수록 사후확률은 **진짜 값에 수렴**한다.

## 예제: 동전을 계속 던지면

공정한 동전인지(H₁, p=0.5) 편향된 동전인지(H₂, p=0.7) 모른다. 한 번씩 던지면서 업데이트해보자.

```
초기 사전: P(H₁) = 0.5, P(H₂) = 0.5

═══ 1번째 던지기: 앞면 ═══

우도:
  P(앞|H₁) = 0.5
  P(앞|H₂) = 0.7

P(H₁|앞) = 0.5 × 0.5 / (0.5×0.5 + 0.7×0.5) = 0.25/0.60 = 0.417
P(H₂|앞) = 0.7 × 0.5 / 0.60 = 0.35/0.60 = 0.583

→ 앞면 1번으로 H₂ 쪽으로 약간 이동

═══ 2번째 던지기: 앞면 ═══

사전을 업데이트된 값으로:
  P(H₁) = 0.417, P(H₂) = 0.583

P(H₁|앞) = 0.5 × 0.417 / (0.5×0.417 + 0.7×0.583) = 0.2085/0.6166 = 0.338
P(H₂|앞) = 0.662

→ 앞면 2연속으로 H₂ 쪽으로 더 이동

═══ 3번째 던지기: 뒷면 ═══

P(H₁|뒤) = 0.5 × 0.338 / (0.5×0.338 + 0.3×0.662) = 0.169/0.368 = 0.459
P(H₂|뒤) = 0.541

→ 뒷면이 나오면 H₁(공정) 쪽으로 되돌아온다!

═══ 10번 던진 결과: 앞 7번, 뒤 3번 ═══

P(H₁) ≈ 0.13
P(H₂) ≈ 0.87

═══ 100번 던진 결과: 앞 52번, 뒤 48번 ═══
(실제로 공정한 동전이었다면)

P(H₁) ≈ 0.98
P(H₂) ≈ 0.02

→ 데이터가 충분하면 진실(공정)에 수렴한다!
```

**이것이 베이지안 학습이다.** 처음에 잘못된 믿음(반반)을 가지고 있어도, 데이터가 쌓이면 진실에 수렴한다.

## 사전확률의 영향은 점점 사라진다

```
사전 A: P(H₁) = 0.9 (공정하다고 강하게 믿음)
사전 B: P(H₁) = 0.1 (편향이라고 강하게 믿음)

같은 데이터(100번 중 앞면 50번)를 관측하면:

사전 A → 사후: P(H₁) ≈ 0.998
사전 B → 사후: P(H₁) ≈ 0.998

시작점이 달라도 충분한 데이터 후에는 거의 같은 곳에 도달한다.
(공정 동전 H₁: p=0.5 vs 편향 H₂: p=0.7 가정.
 50/100 앞면이면 우도비 L(H₁)/L(H₂) ≈ 6126이라
 사전 0.9든 0.1이든 사후가 0.998 부근에 모인다.)
```

이것이 "사전확률이 주관적이어도 괜찮다"는 베이지안의 주장의 근거다. 데이터가 충분하면 사전의 영향은 씻겨 나간다. 하지만 데이터가 적을 때는 사전이 결과를 지배한다 — 그래서 사전 선택이 중요하다.

## 연속 사전분포: 베타 분포 맛보기

지금까지는 가설이 2개(공정 vs 편향)였다. 하지만 현실에서는 "앞면 확률이 정확히 몇인지"를 연속적으로 추정하고 싶다.

```
가설이 2개:     P(p=0.5) vs P(p=0.7)
가설이 무한개:  P(p=0.0), P(p=0.01), ..., P(p=0.99), P(p=1.0)
               → p에 대한 확률분포가 필요
```

이때 사전분포로 **베타 분포(Beta distribution)**를 쓴다.

$$\text{Beta}(\alpha, \beta)$$에서 $$\alpha$$는 "사전에 관측한 앞면 수 + 1", $$\beta$$는 "사전에 관측한 뒷면 수 + 1"이다.

![다양한 파라미터에 따른 베타 분포의 확률밀도함수](/assets/img/posts/bayes/beta_distribution_pdf.svg)
*다양한 α, β 파라미터에 따른 베타 분포 — α=β=1이면 균등분포, 파라미터가 커질수록 분포가 뾰족해진다 (출처: [Wikipedia](https://en.wikipedia.org/wiki/Beta_distribution))*

데이터가 쌓일수록 분포가 **뾰족해진다** — 불확실성이 줄어든다는 뜻이다.

데이터가 쌓일수록 분포가 **뾰족해진다** = 불확실성이 줄어든다. 이것이 [확률 변수](/random-variable/)에서 다룬 확률분포가 베이지안 추론에서 쓰이는 방식이다.

**업데이트 규칙이 놀랍도록 간단하다:**

```
사전: Beta(α, β)
데이터: 앞면 h번, 뒷면 t번
사후: Beta(α + h, β + t)

파라미터에 관측 수를 더하면 끝!
```

이것은 베타 분포가 이항분포의 **켤레 사전분포(conjugate prior)**이기 때문이다. 사전과 사후가 같은 형태의 분포가 되어 계산이 깔끔하다.

# Level 3: 베이지안 사고와 실전

## 빈도주의 vs 베이지안 — 같은 문제, 다른 답

동전을 10번 던져서 7번 앞면이 나왔다. "이 동전의 앞면 확률은?"

**빈도주의 답:**

```
점 추정: p̂ = 7/10 = 0.7  (최대우도추정, MLE)

95% 신뢰구간: (0.35, 0.93)

해석: "진짜 p가 이 구간에 있을 확률이 95%"... 가 아니다!
      "이런 방식으로 구간을 만들면, 100번 중 95번은
       진짜 p를 포함한다"는 뜻이다.

진짜 p는 고정된 값이고, 구간이 랜덤이다.
```

**베이지안 답:**

```
사전: Beta(1, 1) (균등, 아무 정보 없음)
데이터: 앞면 7, 뒷면 3
사후: Beta(8, 4)

사후 평균: 8/(8+4) = 0.667  (MLE의 0.7과 다르다!)

95% 신용구간(credible interval): (0.39, 0.89)

해석: "p가 이 구간에 있을 확률이 95%"
      이것은 진짜로 확률이다. 직관적이다.

p는 불확실한 것이고, 우리의 믿음이 분포로 표현된다.
```

**차이 정리:**

```
              빈도주의                    베이지안
──────────────────────────────────────────────────────
파라미터 p    고정된 미지의 값              불확실한 확률 변수
확률의 의미   장기 빈도                    믿음의 정도
사전 정보    사용 안 함                    사전분포로 반영
결과         점 추정 + 신뢰구간           사후분포
구간 해석    "반복하면 95%가 포함"        "여기에 있을 확률 95%"
데이터 적을 때 불안정                     사전이 안정화
데이터 많을 때 수렴                       수렴 (사전 영향 사라짐)
```

## 베이지안이 유리한 상황

**1. 데이터가 적을 때**

```
신약 임상 시험: 환자 5명 중 4명이 호전.
  빈도주의: p̂ = 0.8, 신뢰구간 (0.28, 0.99)  ← 넓어서 쓸모없음
  베이지안: 기존 유사 약물 데이터를 사전분포로 활용 → 더 좁은 구간
```

**2. 순차적으로 데이터가 들어올 때**

```
스팸 필터: 이메일이 하나씩 들어온다
  빈도주의: 매번 전체 데이터로 모델을 다시 학습
  베이지안: 사후를 사전으로 넘기며 점진적 업데이트 → 효율적
```

**3. 의사결정에 불확실성이 중요할 때**

```
자율주행: "앞에 보행자가 있을 확률"
  빈도주의: 있다/없다 이진 판단
  베이지안: P(보행자) = 0.73 → 확률에 따라 감속 정도 조절

의료: "이 환자가 암일 확률"
  빈도주의: 양성/음성 이진 판단
  베이지안: P(암) = 0.23 → 확률에 따라 추가 검사 결정
```

## 실전 응용: 베이지안 A/B 테스트

웹사이트에서 버튼 색상을 바꿨다. 빨간 버튼(A)과 파란 버튼(B)의 전환율을 비교하고 싶다.

```
데이터:
  A (빨강): 1000명 방문, 50명 클릭 → 5.0%
  B (파랑): 1000명 방문, 62명 클릭 → 6.2%

빈도주의 접근:
  카이제곱 검정 → p-value = 0.21 → "유의하지 않다" (p > 0.05)
  결론: "차이가 있다고 말할 수 없다"
  → 그래서 어쩌라고? B를 쓸지 A를 유지할지 결정할 수 없다.

베이지안 접근:
  사전: Beta(1, 1) for both
  사후 A: Beta(51, 951)   평균 ≈ 5.1%
  사후 B: Beta(63, 939)   평균 ≈ 6.3%

  P(B > A) 계산 (시뮬레이션 또는 적분):
  → P(B가 A보다 나음) ≈ 89%

  결론: "B가 A보다 나을 확률이 89%다.
        11%의 위험을 감수하고 B를 채택할 것인가?"
  → 의사결정자에게 직접적인 정보를 제공
```

## 실전 응용: 베이지안 추천 시스템

평점이 별로 없는 신규 상품의 평균 별점을 추정하고 싶다.

```
문제:
  상품 A: 리뷰 2개, 평균 5.0점
  상품 B: 리뷰 500개, 평균 4.3점
  → 평균만 보면 A가 낫지만, 2개로 5.0은 신뢰할 수 없다

빈도주의: 표본 평균 그대로 사용 → 리뷰 적은 상품이 과대평가됨

베이지안:
  사전: 전체 상품의 평균 평점 μ₀ = 3.8, 신뢰도 C = 50
  (= "리뷰가 없으면 3.8점으로 시작, 50개 이상 쌓여야 사전을 압도")

  사후 평균 = (C × μ₀ + n × x̄) / (C + n)

  상품 A: (50 × 3.8 + 2 × 5.0) / (50 + 2) = 200/52 = 3.85점
  상품 B: (50 × 3.8 + 500 × 4.3) / (50 + 500) = 2340/550 = 4.25점

  → A의 5.0은 3.85로 내려가고 (데이터 부족)
  → B의 4.3은 거의 그대로 4.25 (데이터 충분)
```

이것이 IMDb, Reddit 등에서 사용하는 **베이지안 평균(Bayesian average)**이다. 리뷰가 적은 항목이 상위에 올라가는 것을 방지한다.

## 베이지안의 한계

**1. 사전분포 선택의 주관성**

```
같은 데이터에 대해:
  사전 A: "이 동전은 공정할 것이다" → Beta(100, 100)
  사전 B: "이 동전은 편향일 것이다" → Beta(2, 8)

→ 사후확률이 달라진다.
→ "객관적"이어야 하는 과학에서 주관이 들어가는 것이 불편할 수 있다.
→ 데이터가 충분하면 해소되지만, 적을 때는 사전이 결과를 지배한다.
```

**2. 계산 비용**

```
단순한 경우: Beta-Binomial처럼 공식으로 풀림 (켤레 사전)
복잡한 경우: 적분이 불가능 → MCMC(마르코프 체인 몬테카를로) 필요
            → 수만~수백만 번의 시뮬레이션 → 느리다

딥러닝 + 베이지안: 파라미터가 수백만 개
                   → 각각에 분포를 부여하면 계산량 폭발
                   → 변분 추론(Variational Inference) 등 근사 필요
```

**3. 빈도주의가 더 적합한 경우**

```
- 대규모 데이터: 사전의 영향이 무시할 수준 → 빈도주의와 결과 동일
- 규제/법적 판단: p-value 기반의 빈도주의가 표준
- 단순한 가설 검정: 빈도주의가 더 직관적일 수 있음
```

# Level 4: 베이즈에서 머신러닝/딥러닝으로

베이즈 정리는 "과거의 교과서 공식"이 아니다. 현대 ML/DL의 핵심 개념 상당수가 베이지안 관점에서 자연스럽게 유도된다.

## MAP 추정 — 정규화(Regularization)의 정체

신경망을 학습할 때 **L2 정규화(weight decay)**를 쓴다. 왜 가중치가 작을수록 좋은가? 베이지안으로 보면 명확하다.

```
빈도주의: 최대우도추정 (MLE)
  θ̂ = argmax P(D|θ)
  "데이터를 가장 잘 설명하는 파라미터를 찾아라"

베이지안: 최대사후확률 추정 (MAP)
  θ̂ = argmax P(θ|D) = argmax P(D|θ) · P(θ)
  "데이터도 잘 설명하고, 사전적으로도 그럴듯한 파라미터를 찾아라"
```

로그를 취하면:

```
log P(θ|D) = log P(D|θ) + log P(θ) + const

MLE:  max log P(D|θ)               ← 손실 함수만 최소화
MAP:  max log P(D|θ) + log P(θ)    ← 손실 함수 + 사전분포
```

사전분포로 가우시안 $$P(\theta) = N(0, \sigma^2)$$를 쓰면:

```
log P(θ) = -θ²/(2σ²) + const

→ MAP = max [log P(D|θ) - λθ²]

이것은 정확히:
  Loss = CrossEntropy + λ × ||θ||²
       = 손실 함수    + L2 정규화

L2 정규화 = "파라미터가 0 근처의 가우시안을 따른다"는 사전 믿음!
```

```
라플라스 사전 P(θ) ∝ e^(-|θ|/b) 를 쓰면:

→ MAP = max [log P(D|θ) - λ|θ|]
→ L1 정규화 (Lasso)

L1 = "파라미터가 정확히 0일 수 있다"는 사전 믿음
   → 희소(sparse) 해를 만든다
```

**정리:**

```
정규화 기법       = 베이지안 사전분포
─────────────────────────────────────
L2 (Ridge)       = 가우시안 사전 N(0, σ²)
L1 (Lasso)       = 라플라스 사전
Dropout          = 베이지안 근사 (Gal & Ghahramani, 2016)
Weight Decay     = 가우시안 사전의 MAP 추정
Early Stopping   = 사후분포의 최빈값(mode)에 도달하기 전 멈추기

우리가 "트릭"이라고 생각하던 것들이
사실은 베이지안 프레임워크의 자연스러운 결과다.
```

## 나이브 베이즈 — 가장 단순한 베이지안 분류기

[조건부 확률](/conditional-probability/)에서 스팸 필터를 만들었다. 그 구조를 다시 보면:

```
P(스팸 | 단어들) ∝ P(스팸) × P(단어₁|스팸) × P(단어₂|스팸) × ...

이것은 베이즈 정리 + 조건부 독립 가정이다:
  사후 ∝ 사전 × 우도₁ × 우도₂ × ...

"나이브" = 단어들이 조건부 독립이라는 순진한 가정
  → 현실에서는 "무료"와 "할인"이 함께 나올 확률이 높지만 무시
  → 그래도 놀랍도록 잘 작동함
```

나이브 베이즈가 실전에서 강한 이유:

```
1. 학습이 매우 빠르다 (카운팅만 하면 됨)
2. 데이터가 적어도 작동한다 (사전이 안정화)
3. 고차원에서도 잘 작동한다 (차원의 저주에 강함)
4. 확률을 직접 출력한다 (threshold 조절 가능)

사용처:
  - 스팸 필터 (Gmail 초기 버전)
  - 문서 분류 (뉴스 카테고리 분류)
  - 감성 분석 (긍정/부정 분류)
```

## 베이지안 신경망 — 불확실성을 아는 딥러닝

일반 신경망은 **점 추정**을 한다. 가중치 w = 2.37처럼 하나의 값을 학습한다. 예측할 때도 "고양이입니다(97%)"처럼 하나의 답을 낸다.

문제: **모르는 것을 모른다.**

```
일반 신경망:
  학습 데이터에 없는 이미지 입력
  → "고양이입니다 (99%)"     ← 틀렸는데 확신이 높다!
  → 본 적 없는 입력에도 자신있게 답함

베이지안 신경망:
  같은 이미지 입력
  → "고양이일 수도 있고 개일 수도 있습니다 (불확실)"
  → 학습 데이터와 다른 입력이면 "모른다"고 답할 수 있음
```

**구조:**

```
일반 신경망:
  각 가중치 = 숫자 하나 (w = 2.37)
  → 예측: 하나의 출력

베이지안 신경망:
  각 가중치 = 확률분포 (w ~ N(2.37, 0.15²))
  → 예측: 분포에서 여러 번 샘플링 → 여러 출력 → 평균과 분산

  가중치가 분포이므로:
  분산 작음 → 가중치에 확신 → 예측이 안정적
  분산 큼   → 가중치에 불확신 → 예측이 흔들림 = "모르겠다"
```

**왜 중요한가:**

```
자율주행:
  "앞에 보행자가 있을 확률 95%, 불확실성 낮음" → 브레이크
  "앞에 보행자가 있을 확률 60%, 불확실성 높음" → 감속 + 추가 센서 확인
  → 불확실성을 모르면 60%에서도 브레이크를 밟아 사고 방지 불가

의료 AI:
  "이 X-ray에서 폐렴 확률 80%, 불확실성 낮음" → 진단 보조
  "이 X-ray에서 폐렴 확률 80%, 불확실성 높음" → 전문의 판단 요청
  → 같은 80%라도 불확실성에 따라 행동이 달라야 함
```

## VAE — 생성 모델의 베이지안 구조

VAE(Variational Autoencoder)는 베이지안 추론을 딥러닝에 적용한 대표적인 모델이다.

```
베이즈 정리와 VAE의 대응:

  P(z|x) = P(x|z) · P(z) / P(x)

  z: 잠재 변수 (latent variable) — 데이터의 "핵심 요약"
  x: 관측 데이터 (이미지, 텍스트 등)

  P(z):    사전분포 → 가우시안 N(0, I)로 설정
  P(x|z):  디코더 (z에서 x를 생성)
  P(z|x):  인코더 (x에서 z를 추론) — 직접 계산 불가능!
  P(x):    증거 — 적분 불가능!

  P(x)를 직접 구할 수 없으므로
  → 변분 추론: Q(z|x)로 P(z|x)를 근사
  → ELBO(Evidence Lower Bound)를 최대화
```

```
VAE의 손실 함수:

  Loss = 복원 오차 + KL 발산
       = "입력을 잘 복원하는가" + "잠재 분포가 사전(가우시안)에 가까운가"

  KL 발산 = 사전과 사후의 차이
         = 베이지안의 "사전에서 너무 벗어나지 마라"

이것은 MAP의 정규화와 같은 구조다:
  Loss = 데이터 적합도 + 사전으로부터의 벌점
```

VAE뿐 아니라 **확산 모델(Diffusion Model)**, **흐름 모델(Flow Model)** 등 현대 생성 모델의 상당수가 이 베이지안 프레임워크 위에 세워져 있다.

## 강화학습에서의 베이지안

탐색(exploration)과 착취(exploitation)의 딜레마에서 베이지안이 자연스러운 해를 제공한다.

```
문제: 슬롯머신 3대. 각각의 당첨 확률을 모른다.

빈도주의 접근 (ε-greedy):
  90%: 지금까지 가장 좋았던 머신 선택 (exploitation)
  10%: 랜덤으로 아무거나 선택 (exploration)
  → 고정된 10%를 영원히 낭비

베이지안 접근 (Thompson Sampling):
  각 머신의 당첨 확률에 Beta 사전분포를 부여
  매 라운드:
    1. 각 머신의 사후 분포에서 샘플 하나씩 뽑음
    2. 샘플이 가장 높은 머신 선택
    3. 결과 관측 → 사후 업데이트

  머신 A: Beta(3, 8)  → 샘플 0.21
  머신 B: Beta(5, 5)  → 샘플 0.62  ← 이번에 B 선택
  머신 C: Beta(2, 9)  → 샘플 0.15

  불확실한 머신은 가끔 높은 샘플이 나와서 선택됨 → 자연스러운 탐색
  확실히 나쁜 머신은 낮은 샘플만 나와서 선택 안 됨 → 자연스러운 착취
  → 탐색/착취 비율을 데이터가 자동으로 조절!
```

## 정리

```
Level 1:
  베이즈 정리 = 곱셈 규칙 두 가지를 등호로 연결한 것
  사후 = (우도 × 사전) / 증거
  조건부 확률의 방향을 뒤집는 도구

Level 2:
  사후가 다음의 사전이 된다 → 반복 업데이트
  데이터가 쌓이면 사전의 영향은 사라지고 진실에 수렴
  연속 추정은 베타 분포 → 파라미터에 관측 수를 더하면 끝

Level 3:
  빈도주의 = "p는 고정, 데이터가 랜덤"
  베이지안 = "데이터는 고정, p가 랜덤(불확실)"
  데이터 적을 때, 순차 업데이트, 불확실성 표현에서 베이지안이 강하다

Level 4:
  L2 정규화 = 가우시안 사전의 MAP 추정
  나이브 베이즈 = 조건부 독립 + 베이즈 정리
  베이지안 신경망 = "모르는 것을 안다" (불확실성 추정)
  VAE = 변분 추론으로 사후분포를 근사
  Thompson Sampling = 사후 분포에서 샘플링으로 탐색/착취 자동 조절
```

핵심 개념을 정리하면 다음과 같다. 베이즈 정리는 $$P(H \mid D) = \frac{P(D \mid H) \cdot P(H)}{P(D)}$$이며, 조건부 확률의 방향을 전환하는 도구다. 반복 업데이트는 사후₁ → 사전₂ → 사후₂ → ... 의 과정으로, 데이터가 쌓이면 진실에 수렴한다. 베타-이항 모델에서는 사전 Beta(α,β)에 데이터(앞면 h번, 뒷면 t번)를 관측하면 사후가 Beta(α+h, β+t)가 되며, 이것이 켤레 사전의 위력이다.
