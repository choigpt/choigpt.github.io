  3. 개인 피드 쿼리 최적화 계획

  현재 구조 (병목 원인)

  4. findAccessibleClubIds(userId)  → user_club에서 소속 클럽 ID 리스트 (캐시 5분)

  기하급수적으로 느려짐

  방안 1: 복합 인덱스 추가 (가장 빠른 효과)

  - 현재 accessibleClubIds만 캐시, pass1 결과도 Redis 캐시 (TTL 30s~1m)

● 문제 원인: MySQL 데이터 이중 인코딩 (Double Encoding)                                                                                                                                                           
                                                                                                                                                                                                                  
  핵심 문제                                                                                                                                                                                                       
                                                                                                                                                                                                                  
  seed 데이터가 MySQL에 삽입될 때 **클라이언트 charset이 latin1**인 상태로 UTF-8 바이트가 전송됨.                                                                                                                 
                                                                                                                                                                                                                  
  발생 과정                                                                                                                                                                                                       
                                                                                                                                                                                                                  
  1. seed SQL의 "축구동호회" → UTF-8 바이트: EC B6 95 EA B5 AC ...                                                                                                                                                
  2. MySQL 클라이언트(Docker내 mysql CLI)의 기본 charset: latin1                                                                                                                                                  
  3. MySQL은 이 UTF-8 바이트를 latin1 문자로 해석                                                                                                                                                                 
     EC → ì(U+00EC), B6 → ¶(U+00B6), 95 → •(U+2022) ...                                                                                                                                                           
  4. 컬럼이 utf8mb4이므로 이 latin1 문자들을 다시 UTF-8로 인코딩하여 저장                                                                                                                                         
     ì → C3 AC, ¶ → C2 B6, • → E2 80 A2 ...                                                                                                                                                                       
                                                                                                                                                                                                                  
  결과: 축구동호회 1이 17바이트, 7글자여야 하는데 → 34바이트, 17글자로 저장됨.                                                                                                                                    
                                                                                                                                                                                                                  
  왜 MySQL CLI에서는 검색이 됐는가?                                                                                                                                                                               
                                                                                                                                                                                                                  
  - Docker 내 mysql CLI도 latin1 기본 → 검색어도 같은 방식으로 이중 인코딩 → 매칭 성공                                                                                                                            
  - JDBC (utf8mb4 연결) → 검색어가 정상 UTF-8로 전송 → 이중 인코딩된 데이터와 매칭 안 됨 → 0건                                                                                                                    
                                                                                                                                                                                                                  
  수정                                                                                                                                                                                                            
                                                                                                                                                                                                                  
  -- 이중 인코딩 역변환 (utf8mb4 → latin1 → binary → utf8mb4)                                                                                                                                                     
  ALTER TABLE club MODIFY name VARCHAR(255) CHARACTER SET latin1;                                                                                                                                                 
  ALTER TABLE club MODIFY name VARBINARY(255);                                                                                                                                                                    
  ALTER TABLE club MODIFY name VARCHAR(255) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;                                                                                                                     
  -- FULLTEXT 인덱스 재생성
  ALTER TABLE club ADD FULLTEXT INDEX ft_club_search (name, description) WITH PARSER ngram;
